## 목적

에디터는 복잡한 문서 편집기가 아니라, 산만함을 줄이고 에세이 작성에 집중하게 만드는 작업 공간이어야 합니다.

## 요구사항

- 모바일 우선 레이아웃
- 산만한 도구 노출 최소화
- 초고 작성과 퇴고 흐름 분리
- 자동 저장과 버전 관리
- 하이라이트와 메모 연결

## 경험 원칙

- 집필 중에는 입력 흐름을 끊지 않는다.
- 퇴고 기능은 작성 단계와 분리된 검토 모드로 제공한다.
- 장문의 구조를 다루기 어려운 모바일 환경을 보완해야 한다.

## 현재 구현 단서

현재 웹 앱은 TipTap 기반 편집기 의존성을 포함하고 있으며, 이후 에디터 경험은 이 기반 위에서 확장될 가능성이 높습니다.
