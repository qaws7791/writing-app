## 1. Executive Summary

**글필(Geulpil)**은 한국어 사용자를 위한 에세이 글쓰기 학습 플랫폼이다. "여정(Journey)"이라는 구조화된 커리큘럼을 통해 사용자가 단계별로 사고력과 글쓰기 역량을 성장시키도록 설계되었다.

기존 글쓰기 앱들이 "습관 형성(분량 채우기)"이나 "AI 대필"에 머무는 반면, 글필은 Quill.org의 즉각적 AI 피드백, NoRedInk의 관심사 기반 개인화, Write & Improve with Cambridge의 제출-피드백-수정 반복 루프를 레퍼런스로 삼아, **"글쓰기 과정 자체를 통한 사고력 훈련"**이라는 차별화된 가치를 제공한다.

핵심 설계 원칙은 세 가지이다. 첫째, AI는 대필이 아닌 소크라테스식 코칭을 수행한다. 둘째, 모든 글쓰기는 초안-피드백-수정의 다단계 워크플로우를 거친다. 셋째, 사용자의 인지적 성장을 데이터로 시각화하여 리텐션을 확보한다.

## 2. 문제 정의

### 2.1. 시대적 맥락

숏폼 콘텐츠와 알고리즘 기반 소비 문화는 주의 집중 시간을 단축하고, 비판적 사고보다 감정 반응을 우선하는 미디어 소비 습관을 형성했다. 글쓰기의 사회적 필요가 감소하면서(편지, 일기, 보고서 → 메신저, 이모지, 음성 메시지), 논리적 구성 능력과 표현력이 전반적으로 저하되고 있다. 생성형 AI 도구의 남용은 "모름 → 질문 → AI가 즉답"이라는 단축 경로를 제공하여, 스스로 사유하는 과정 자체를 건너뛰게 만든다.

### 2.2. 사용자가 겪는 구체적 고통

| 고통 영역 | 핵심 문제 | 결과 |
|---|---|---|
| **아이디어 생성** | 무엇을 써야 할지 막막함, 소재/주제 생성 어려움 | 글쓰기 시작 자체를 회피 |
| **구성·구조화** | 서론-본론-결론의 전체 틀과 문단 구성에 대한 불안 | 비구조적이고 산만한 글 |
| **완벽주의·미루기** | 초고를 "형편없다"고 판단하여 반복 갈아엎음 | 품질/만족도 저하, 미완성 |
| **퇴고 어려움** | 초고-퇴고-버전 관리가 어렵고 "나중에 고치기"가 무한 대기열 | 성장 없는 반복 |
| **학습 결핍** | 에세이 구조, 퇴고 전략에 대한 체계적 학습 기회 부재 | 실력 정체 |
| **공개 불안** | "내 글이 틀릴까", 타인의 평가에 대한 자기검열 | 글쓰기 중단, 침묵 효과 |
| **AI 의존** | AI 생성 텍스트의 획일화, 개성 약화, 사고력 퇴화 | 비판적 사고 근육 상실 |

### 2.3. 기존 도구의 한계

**생산성 앱의 한계:** 정보를 수집·분류하는 행위 자체를 "사고했다"고 착각하게 만드는 수집가의 오류(Collector's Fallacy). 정보의 단순 축적은 통찰로 이어지지 않는다.

**모바일 앱의 한계:** 한 번에 보이는 정보량이 적어 전체 맥락 파악이 어렵고, 앱 알림과 메신저로 인해 글쓰기 흐름이 자주 끊긴다. 비선형적 사고(메모 → 순서 변경 → 묶기 → 나누기)를 지원하는 도구가 부족하다.

**생성형 AI의 한계:** 허위·환각 정보 생산, 문체 획일화, 저자 개성 약화, 과도한 의존에 의한 스킬 저하, "AI 티 나는 글"에 대한 거부감 등 근본적인 문제가 존재한다.

## 4. 레퍼런스 서비스 분석

### 4.1. Quill.org — 즉각적 AI 피드백 + 문장 단위 훈련

Quill은 1,000개 이상의 차별화된 활동을 제공하며, 학생이 문장을 제출하면 AI가 즉시 피드백을 제공한다. 핵심 도구는 6가지(Diagnostics, Connect, Grammar, Proofreader, Lessons, Reading for Evidence)이며, 활동당 10~15분의 짧은 세션 구조를 채택한다. 기술적 문법 용어 대신 기능 중심 설명을 사용하고, 수정을 글쓰기의 자연스러운 과정으로 격려한다.

**글필에의 시사점:** 짧은 세션 구조, 즉각적 AI 피드백, "정답/오답"이 아닌 코칭형 피드백 톤, 진단 → 맞춤 학습 계획 생성 구조를 벤치마킹한다.

### 4.2. NoRedInk — 관심사 기반 개인화 + 적응형 학습

NoRedInk는 학생의 관심사(좋아하는 영화, 친구 이름 등)를 활용하여 연습 문장을 개인화한다. AI 채점 보조(Grading Assistant)로 교사의 채점 시간을 40% 단축하고, 학생에게 2배 더 많은 피드백을 제공한다. 미국 학교의 60% 이상이 사용 중이다.

**글필에의 시사점:** 온보딩 시 사용자 관심사 수집 → 글감/프롬프트 개인화, 적응형 난이도 조절, 관심사 기반 몰입감을 차용한다.

### 4.3. Write & Improve with Cambridge — 제출-피드백-수정 루프

Write & Improve는 CEFR 기준으로 글쓰기 수준을 평가하고, 세 가지 유형의 피드백(총괄적 점수, 간접적 반교정 피드백, 격려 중심 개관 피드백)을 제공한다. 90% 이상 확신할 때만 오류를 지적하며, 한 번에 너무 많은 피드백을 주지 않도록 설계되었다. 수정 후 재제출하면 새로운 피드백이 제공되는 반복 루프가 핵심이다.

**글필에의 시사점:** 수준별 자동 평가, "제출 → 피드백 → 수정 → 재제출" 루프, 점진적 피드백 노출(한 번에 모든 오류를 보여주지 않음), 격려 중심 톤을 차용한다.

## 5. 타겟 사용자

### 5.1. 주요 페르소나

**페르소나 A — "글쓰기 입문자" 민지 (22세, 대학생)**

- 에세이 과제에서 늘 막막함을 느끼고, 구조를 잡는 방법을 모른다.
- AI에 의존해 과제를 제출하지만, 실력이 늘지 않는 것을 인지하고 있다.
- 원하는 것: 단계별로 글 쓰는 법을 배우고, 내 실력이 올라가는 것을 확인하고 싶다.
- 핵심 동기: 학업 성취, 취업 준비 포트폴리오

**페르소나 B — "글쓰기 재개자" 준혁 (31세, 직장인)**

- 브런치에 글을 써보고 싶지만, 작가 승인 절차와 공개 부담이 진입장벽이다.
- 업무 보고서 외에 개인적인 에세이를 써본 적이 없다.
- 원하는 것: 부담 없이 시작하고, 조금씩 나만의 글쓰기 스타일을 찾고 싶다.
- 핵심 동기: 자아 성찰, 정체성 표현, 사이드 프로젝트

**페르소나 C — "글쓰기 성장자" 지원 (27세, 주니어 마케터)**

- 블로그를 운영 중이지만, 글의 논리적 깊이와 구조에 한계를 느낀다.
- 피드백을 받을 곳이 없어 혼자 정체되어 있다.
- 원하는 것: 내 글의 구조적 약점을 진단받고, 체계적으로 개선하고 싶다.
- 핵심 동기: 커리어 성장, 전문성 구축

### 5.2. 타겟 세그먼트 우선순위

1. **1차 타겟:** 20~30대 글쓰기에 관심 있지만 시작을 못 하는 "잠재 작가"
2. **2차 타겟:** 대학생 (에세이·리포트 역량 향상 필요)
3. **3차 타겟:** 직장인 (업무 커뮤니케이션 및 사이드 프로젝트)

## 6. 제품 비전 및 전략

### 6.1. 제품 비전

> **"글쓰기를 통해 생각하는 힘을 기르는 여정(Journey)."**

글필은 AI가 글을 대신 써주는 도구가 아니라, 사용자가 **스스로 생각을 구조화하고, 피드백을 통해 사고를 확장하며, 자신의 인지적 성장을 체감하는** 학습 플랫폼이다.

### 6.2. 핵심 가치 제안 (Value Proposition)

| 기존 경험 | 글필의 제안 |
|---|---|
| 빈 화면 앞에서 막막함 | 글감(Writing Prompt)이 사고를 촉발 |
| 한 번 쓰고 끝나는 일회성 글쓰기 | 초안 → AI 피드백 → 수정의 다단계 워크플로우 |
| 혼자 쓰고 혼자 판단 | 소크라테스식 AI 코칭 + 커뮤니티 피드백 |
| 실력이 느는지 알 수 없음 | 사고력 성장 대시보드로 인지적 발달 시각화 |
| AI가 대신 써줌 → 실력 퇴화 | AI는 질문자 역할, 사고의 주체는 사용자 |

### 6.3. 설계 원칙

1. **사고 우선 (Thinking First):** 모든 기능 설계의 기준은 "이것이 사용자의 사고를 자극하는가?"이다.
2. **점진적 노출 (Progressive Disclosure):** 한 번에 모든 것을 보여주지 않는다. 피드백도, 학습 단계도, 기능도 사용자의 수준에 맞춰 점진적으로 노출한다.
3. **수정은 성장이다 (Revision = Growth):** 초고의 완벽함을 추구하지 않는다. 수정 과정 자체를 긍정적 경험으로 설계한다.
4. **간결함 (Simplicity):** 복잡함보다 간결함. 1인 개발 프로젝트의 제약을 인정하고, 핵심 루프에 집중한다.
5. **모바일 네이티브:** 엄지 최적화 UX, 짧고 반복 가능한 세션 구조.

## 7. 성공 조건

### 7.1. 서비스 전반의 성공 조건

| 조건 | 글필에의 적용 |
|---|---|
| 문제-해결 적합성 (PSF) | "글쓰기 교육 × 게이미피케이션" 공백 시장을 정확히 공략 |
| 압도적 UX + 낮은 인지 비용 | 글감 → 글쓰기 → 피드백의 3스텝 핵심 루프를 단순화 |
| 개인화가 기본값 | 온보딩 시 관심사/수준 파악 → 글감·여정·피드백 개인화 |
| 강력한 리텐션 메커니즘 | 여정 진행률, 스트릭, 사고력 성장 대시보드 |
| 명확한 수익 모델 | Free(기본 글감 + 제한된 여정) / Pro(전체 여정 + 고급 AI 코칭) |
| 빠른 실험 + 지속적 진화 | 1인 개발에 최적화된 린(Lean) 프로세스 |

### 7.2. 젊은 세대 타겟 성공 조건

| 조건 | 글필에의 적용 |
|---|---|
| 즉각적 도파민 구조 | 글 제출 → 즉시 AI 피드백 → 수준 점수 변화 확인 |
| 정체성 표현 도구 | "나의 여정" 프로필, 글 아카이브가 곧 사고의 포트폴리오 |
| 진입 장벽 제로 | 앱 설치 → 글감 선택 → 바로 쓰기 시작 (회원가입 후순위) |
| 게이미피케이션 | 여정 진행률, 세션 달성, 스트릭, 레벨 시스템 |
| 모바일 네이티브 UX | 엄지 최적화, 세션당 10~15분 완결형 |
| AI 기반 개인화 | 글감 추천, 피드백 수준, 여정 추천 모두 AI 기반 |

## 8. 핵심 개념 모델

### 8.1. 여정-세션-스텝 계층 구조

```
여정 (Journey)
├── 세션 1 (Session) — 학습 노드
│   ├── 스텝 1 (Step) — 개념 학습 / 예시 읽기
│   ├── 스텝 2 (Step) — 가이드 질문에 답하기
│   ├── 스텝 3 (Step) — 짧은 글쓰기 연습
│   └── 스텝 4 (Step) — AI 피드백 + 수정
├── 세션 2 (Session) — 학습 노드
│   ├── 스텝 1 (Step) — ...
│   └── ...
├── ...
└── 세션 N (Session) — 최종 에세이 작성 + 종합 피드백
```

### 8.2. 핵심 엔티티 정의

| 엔티티 | 정의 | 예시 |
|---|---|---|
| **글감 (Writing Prompt)** | 사용자의 사고를 촉발하는 질문·주제·상황 카드 | "먼지 쌓인 서랍을 열었을 때, 당신의 시선을 멈추게 한 물건은?" |
| **여정 (Journey)** | 특정 글쓰기 역량을 목표로 한 구조화된 커리큘럼. 여러 세션으로 구성 | "내 주장을 설득력 있게" (7개 세션, 논증 에세이) |
| **세션 (Session)** | 여정 내 하나의 학습 노드. 여러 스텝으로 구성. 10~20분 완결형 | "근거의 구조화" — 반론 대비 연습 |
| **스텝 (Step)** | 세션 내 하나의 마이크로 활동 단위 | 개념 카드 읽기 / 가이드 질문 답변 / 글쓰기 / 피드백 확인 |
| **글 (Writing)** | 사용자가 작성한 텍스트. 글감 또는 세션에 연결 | 에세이 초안, 수정본 |
| **서재 (Library)** | 사용자가 작성한 모든 글의 아카이브 | — |

### 8.3. 글감 유형 분류

| 유형 | 설명 | 메타인지 목표 |
|---|---|---|
| **감각 (Sensory)** | 감각적 경험에서 출발하는 자유 글쓰기 | 관찰력, 표현력 |
| **회고 (Reflection)** | 과거 경험을 되돌아보는 성찰적 글쓰기 | 자기 인식, 의미 구성 |
| **의견 (Opinion)** | 특정 주제에 대한 입장을 논증하는 글쓰기 | 논리적 사고, 근거 제시 |

### 8.4. 여정 카테고리

| 카테고리 | 설명 | 예시 여정 |
|---|---|---|
| **글쓰기 기술** | 에세이 구조, 문단 구성, 퇴고 전략 등 기술적 역량 | "첫 문장의 힘", "설득하는 글쓰기" |
| **마음챙김** | 감정 탐색, 자기 성찰, 일상 관찰 중심 | "새벽의 대화", "나를 찾는 여행" |
| **기술 (Skill)** | 특정 직무·상황 맞춤형 글쓰기 | "업무 이메일 글쓰기", "자기소개서" |

## 9. 정보 구조 및 화면 설계

### 9.1. 글로벌 네비게이션

앱은 하단 탭 바(Bottom Tab Bar)로 4개의 주요 섹션을 제공한다.

```
[ 홈 ]  [ 나의 여정 ]  [ 서재 ]  [ 프로필 ]
```

### 9.2. 화면 상세 설계

---

#### 9.2.1. 홈 (Home)

**화면 구성:**

홈 화면 상단에는 3개의 수평 탭(홈 / 글감 / 여정)이 위치한다.

**홈 탭:**

- **인사 영역:** "좋은 아침입니다, {사용자명}님." 시간대별 동적 인사
- **오늘의 추천 글감 카드:** 매일 갱신되는 글감 1개를 강조 표시. 카드에는 글감 제목과 현재까지의 응답 수가 표시됨
- **현재 진행 중인 여정:** 진행 중인 여정 목록을 카드 형태로 표시. 각 카드에는 여정명, 부제, 썸네일, 진행률(프로그레스 바 + 퍼센트)이 포함됨
- **"더보기" 링크:** 나의 여정 탭으로 이동

**글감 탭:**

- **필터 칩:** 전체 / 감각 / 회고 / 의견 카테고리 필터
- **글감 카드 리스트:** 각 카드에는 브랜드/파트너 로고(선택), 날짜, 유형 태그, 글감 제목, 응답 수, 북마크 버튼이 포함됨
- **스크롤:** 무한 스크롤 또는 페이지네이션

**여정 탭:**

- **필터 칩:** 전체 / 글쓰기 기술 / 마음챙김 / 기술
- **여정 카드 리스트:** 각 카드에는 썸네일, 여정명, 설명 텍스트, 세션 수가 포함됨

---

#### 9.2.2. 글감 상세 (Prompt Detail)

**화면 구성:**

- **헤더:** 뒤로 가기 + 글감 제목 + 더보기(⋮) 메뉴
- **글감 제목:** 대형 타이포그래피로 표시
- **글감 본문:** 사고를 촉발하는 질문 또는 상황 설명
- **참여 통계:** 참여 작가 수 / 작성된 수필 수
- **작성된 글 리스트:** 해당 글감에 대한 다른 사용자(또는 자신)의 글 목록. 각 항목에 날짜, 단어 수 태그, 글 제목, 본문 미리보기

---

#### 9.2.3. 글 에디터 (Writing Editor)

**화면 구성:**

- **헤더:** 뒤로 가기 + 글 제목(편집 가능) + 더보기(⋮) + 완료(✓) 버튼
- **날짜 표시:** 작성일
- **제목 필드:** 큰 폰트로 편집 가능한 제목
- **본문 영역:** 충분한 여백의 클린한 텍스트 입력 영역. 산세리프 본문 폰트, 적절한 줄 간격
- **에디터 기능:** 기본 텍스트 입력 (v1은 최소 기능, 볼드/이탤릭 정도)

---

#### 9.2.4. 글 상세 / 읽기 뷰 (Writing Detail)

**화면 구성:**

- **헤더:** 뒤로 가기 + 글 제목 + 더보기(⋮)
- **날짜:** 작성일
- **제목:** 큰 타이포그래피
- **저자명:** 우측 정렬
- **Writing Prompt 접힘 카드:** 이 글이 연결된 글감 정보를 접힘/펼침(Accordion) UI로 표시. 펼치면 글감 본문과 "글감 상세 보기 →" 버튼이 나타남
- **본문:** 에세이 본문 전체

---

#### 9.2.5. 나의 여정 (My Journeys)

**화면 구성:**

- **페이지 타이틀:** "여정"
- **진행 중인 여정 섹션:** 카드 형태로 현재 참여 중인 여정 목록 표시. 각 카드에는 썸네일, 여정명, 부제(유형 설명), 진행률(프로그레스 바 + 퍼센트)
- **여정 찾기 섹션:** "더보기" 링크 포함. 카테고리별 여정을 2열 그리드 카드로 표시. 각 카드에는 아이콘, 카테고리명, 포함된 테마 수

---

#### 9.2.6. 여정 상세 (Journey Detail)

**화면 구성:**

- **헤더:** 뒤로 가기 + 여정 제목 + 더보기(⋮)
- **히어로 이미지:** 여정 대표 이미지
- **여정 제목:** 큰 타이포그래피
- **여정 설명:** 이 여정에서 다루는 내용과 기대 성과
- **전체 진행률 카드:** "전체 진행률 / {완료 수} / {전체 세션 수} 세션 / {달성률}% 달성" + 세션별 미니 프로그레스 바
- **세션 리스트:** 수직 리스트로 세션을 순서대로 표시. 각 세션 카드에는 상태 아이콘(✓ 완료 / 숫자 진행 중 / 🔒 대기 중), 세션 제목, 세션 설명(진행 중인 경우만), "지금 시작하기 →" 버튼(진행 중인 경우만) 포함. 이전 세션을 완료해야 다음 세션이 잠금 해제되는 선형 진행 구조

---

#### 9.2.7. 세션 진행 (Session Flow)

**모바일 웹 키보드 대응:**

- 글쓰기(WRITE) 스텝에서 키보드 활성화 시:
  - 상단 프로그레스 바는 position: sticky 유지
  - "제출" 버튼은 키보드 위 고정 (visualViewport API 활용)
  - Writing Prompt 참조 카드는 접힘 상태로 최소화하여 입력 영역 확보
- CSS 높이 기준: `height: 100dvh` 사용 (dynamic viewport height, iOS 15.4+/Android Chrome 지원)

**화면 구성 (멀티 스텝):**

세션 진행은 전체 화면을 사용하는 스텝 바이 스텝 플로우이다.

- **상단 프로그레스 바:** 현재 스텝 / 전체 스텝 수 시각화
- **스텝 유형별 UI:**

| 스텝 유형 | UI 구성 |
|---|---|
| **개념 학습 (Learn)** | 제목 + 핵심 개념 설명 텍스트 + 예시 카드 + "다음" 버튼 |
| **예시 읽기 (Read)** | 모범 에세이 또는 문단 + 분석 포인트 하이라이트 + "다음" 버튼 |
| **가이드 질문 (Guided Question)** | 메타인지 촉발 질문 + 텍스트 입력 필드 + "답변 제출" 버튼 |
| **글쓰기 (Write)** | 글감/프롬프트 표시 + 풀스크린 에디터 + 단어 수 카운터 + "제출" 버튼 |
| **AI 피드백 (Feedback)** | 제출된 글 + AI 피드백 카드(강점/개선점/질문) + "수정하기" 또는 "다음" 버튼 |
| **수정 (Revise)** | 피드백이 인라인 표시된 에디터 + "재제출" 버튼 |

---

#### 9.2.8. 서재 (Library)

**화면 구성:**

- **페이지 타이틀:** "글쓰기"
- **검색 바:** "기록된 생각을 검색해보세요"
- **글 목록:** 카드 리스트. 각 카드에는 날짜, 글 제목, 본문 미리보기, 단어 수, 더보기(⋮) 메뉴
- **정렬:** 최신순 (기본)

---

#### 9.2.9. 프로필 / 설정 (Profile / Settings)

**화면 구성:**

- **페이지 타이틀:** "설정"
- **프로필 영역:** 아바타, 사용자명, 이메일
- **통계 카드:** 완료한 여정 수 / 작성한 글 수
- **계정 관리:** 계정 보안, 알림 설정
- **접근성:** 글꼴 크기 (보통/크게/매우 크게), 동작 줄이기 모드
- **서비스 및 데이터:** 구독 관리 (Pro), 데이터 내보내기
- **로그아웃**

## 10. 기능 요구사항 (EARS 표기법)

> EARS (Easy Approach to Requirements Syntax) 표기법을 사용한다.  
> 패턴: Ubiquitous / Event-Driven / Unwanted Behavior / State-Driven / Optional

### 10.1. 온보딩 및 계정

| ID | 유형 | 요구사항 |
|---|---|---|
| REQ-ACC-001 | Event-Driven | **WHEN** 사용자가 소셜 로그인(Google/Kakao) 버튼을 탭하면, **THE SYSTEM SHALL** OAuth 인증을 수행하고 계정을 생성 또는 연결한다. |
| REQ-ACC-002 | Event-Driven | **WHEN** 사용자가 "데이터 내보내기"를 요청하면, **THE SYSTEM SHALL** 사용자의 모든 글을 마크다운 파일로 패키징하여 다운로드 링크를 제공한다. |

### 10.2. 홈

| ID | 유형 | 요구사항 |
|---|---|---|
| REQ-HOME-001 | State-Driven | **WHILE** 사용자가 진행 중인 여정이 있는 상태에서, **THE SYSTEM SHALL** 홈 탭에 진행 중인 여정 카드(여정명, 진행률)를 표시한다. |
| REQ-HOME-002 | Ubiquitous | **THE SYSTEM SHALL** 매일 오전 6시에 사용자의 관심사와 글쓰기 수준에 기반하여 오늘의 추천 글감 1개를 선정한다. |
| REQ-HOME-003 | Event-Driven | **WHEN** 사용자가 홈/글감/여정 탭을 전환하면, **THE SYSTEM SHALL** 0.3초 이내에 해당 탭의 콘텐츠를 표시한다. |

### 10.3. 글감 (Writing Prompt)

| ID | 유형 | 요구사항 |
|---|---|---|
| REQ-PRM-001 | Ubiquitous | **THE SYSTEM SHALL** 글감을 감각/회고/의견의 3가지 유형으로 분류하고, 각 유형에 해당하는 필터 칩을 제공한다. |
| REQ-PRM-002 | Event-Driven | **WHEN** 사용자가 글감 카드를 탭하면, **THE SYSTEM SHALL** 글감 상세 화면(글감 본문, 참여 통계, 관련 글 목록)을 표시한다. |
| REQ-PRM-003 | Event-Driven | **WHEN** 사용자가 글감 상세에서 "글쓰기 시작" 버튼을 탭하면, **THE SYSTEM SHALL** 해당 글감이 연결된 글 에디터를 열고, Writing Prompt 정보를 에디터 상단에 참조용으로 표시한다. |
| REQ-PRM-004 | Event-Driven | **WHEN** 사용자가 글감의 북마크(🔖) 아이콘을 탭하면, **THE SYSTEM SHALL** 해당 글감을 사용자의 북마크 목록에 추가/제거한다. |
| REQ-PRM-005 | Ubiquitous | **THE SYSTEM SHALL** 각 글감 카드에 현재까지의 응답 수를 실시간으로 표시한다. |

### 10.4. 여정 (Journey)

| ID | 유형 | 요구사항 |
|---|---|---|
| REQ-JRN-001 | Ubiquitous | **THE SYSTEM SHALL** 여정을 글쓰기 기술/마음챙김/기술의 3가지 카테고리로 분류하고, 각 카테고리에 해당하는 필터 칩을 제공한다. |
| REQ-JRN-002 | Event-Driven | **WHEN** 사용자가 여정 카드를 탭하면, **THE SYSTEM SHALL** 여정 상세 화면(설명, 진행률, 세션 리스트)을 표시한다. |
| REQ-JRN-003 | Ubiquitous | **THE SYSTEM SHALL** 여정 내 세션들을 선형 진행 구조로 배치하며, 이전 세션이 완료되어야 다음 세션이 잠금 해제된다. |
| REQ-JRN-004 | State-Driven | **WHILE** 세션이 "대기 중" 상태인 동안, **THE SYSTEM SHALL** 해당 세션 카드에 잠금(🔒) 아이콘을 표시하고 탭 상호작용을 비활성화한다. |
| REQ-JRN-005 | Event-Driven | **WHEN** 사용자가 "지금 시작하기" 버튼을 탭하면, **THE SYSTEM SHALL** 해당 세션의 첫 번째 스텝을 전체 화면 플로우로 시작한다. |

```gherkin
    Given 사용자가 설정 화면에 있다
    When 사용자가 "데이터 내보내기"를 탭한다
    Then 확인 다이얼로그가 표시된다
    And 사용자가 확인하면 모든 글이 마크다운 파일로 변환되어 다운로드된다
```

## 12. AI 통합 전략

### 12.1. 핵심 원칙: 소크라테스식 코칭

AI는 **완성된 텍스트를 생성하지 않는다.** 대신 다음의 역할을 수행한다.

| 역할 | 설명 | 예시 |
|---|---|---|
| **진단자** | 글의 구조적/논리적 특성을 분석 | "이 글은 3개의 문단으로 구성되어 있고, 중심 논지가 두 번째 문단에서 전환됩니다." |
| **질문자** | 사고를 확장하는 질문 제시 | "이 주장에 동의하지 않는 사람은 어떤 반론을 펼칠까요?" |
| **격려자** | 개선된 부분을 구체적으로 칭찬 | "초안보다 근거가 구체적이 되었고, 특히 두 번째 예시가 설득력을 높였습니다." |
| **안내자** | 다음 단계를 위한 힌트 제공 | "결론에서 서론의 질문을 다시 한번 떠올려 보는 건 어떨까요?" |

### 12.2. 피드백 구조

```
AI 피드백 카드
├── 💚 강점 (1~2개)
│   └── 구체적으로 잘된 부분과 이유
├── 💡 개선점 (1~2개)
│   └── 문제 지적 + 방향 힌트 (답을 주지 않음)
└── ❓ 사고 촉발 질문 (1개)
    └── 글의 깊이를 확장하는 메타인지 질문
```

### 12.3. 수준별 피드백 전략

| 수준 | 강점 : 개선점 비율 | 피드백 깊이 | 메타인지 질문 유형 |
|---|---|---|---|
| 입문 | 2 : 1 | 문장/문단 단위 | "이 문장에서 가장 하고 싶은 말은?" |
| 중급 | 1 : 1 | 논리 구조 단위 | "이 근거가 주장을 충분히 뒷받침하나요?" |
| 상급 | 1 : 2 | 에세이 전체 단위 | "이 글의 전제에 동의하지 않는 독자를 어떻게 설득하시겠어요?" |

### 12.4. AI가 하지 않는 것

- 완성된 문장이나 문단을 생성하지 않는다.
- 대체 표현을 직접 제시하지 않는다.
- 맞춤법/문법 교정을 자동으로 적용하지 않는다 (지적은 하되, 수정은 사용자 몫).
- 글의 주제나 방향을 결정하지 않는다.

## 13. 게이미피케이션 설계

### 13.1. 리텐션 메커니즘

| 메커니즘 | 설명 | 구현 우선순위 |
|---|---|---|
| **여정 진행률** | 세션별/여정별 프로그레스 바와 달성률 | P0 (MVP) |
| **글쓰기 스트릭** | 연속 글쓰기 일수 추적 | P1 |
| **단어 수 누적** | 총 작성 단어 수 카운터 | P1 |
| **세션 달성 보상** | 세션 완료 시 시각적 축하 애니메이션 | P0 (MVP) |
| **여정 완료 뱃지** | 여정 완료 시 프로필에 뱃지 추가 | P2 |

### 13.2. 동기 부여 전략

**내재적 동기 (Intrinsic):** AI 피드백을 통한 실력 향상 체감, 자신의 글 아카이브 축적, 메타인지 질문을 통한 지적 자극.

**외재적 동기 (Extrinsic):** 여정 진행률과 달성률, 스트릭 시각화, 글감 응답 수를 통한 사회적 참여감.
