# API

## 현재 상태

API 앱은 `apps/api`에 위치하며 Hono를 사용합니다. 현재 패키지 설정 기준으로는 최소 실행 골격에 가깝고, 세부 도메인 구조는 아직 문서화된 상태가 아닙니다.

## 책임

- HTTP 엔드포인트 제공
- 인증, 권한, 데이터 접근을 위한 서버 측 진입점
- 웹 앱과의 계약 유지

## 향후 방향

기존 계획 문서에서는 다음과 같은 확장 방향이 제안되어 있습니다.

- OpenAPI 중심 계약 관리
- 타입 안전한 클라이언트 생성
- 인증과 데이터 계층 통합

## 문서화 기준

- 실제 API 계약이 생기면 엔드포인트 분류와 인증 정책을 이 문서 또는 하위 문서로 분리합니다.
- 구현 전 계획은 명확히 `planned` 성격으로 구분합니다.
