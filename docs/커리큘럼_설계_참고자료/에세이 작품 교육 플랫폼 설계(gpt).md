아래처럼 설계하는 것이 맞습니다. 결론부터 말하면, 한국어 에세이 교육 플랫폼은 **‘에세이 작성기’가 아니라, 읽기→생각하기→짧게 쓰기→문단 쓰기→에세이 쓰기→피드백→고쳐쓰기→출판/성찰**의 전 과정을 쪼갠 **활동 기반 학습 시스템**이어야 합니다. 2022 개정 국어과 교육과정은 기초 문해력, 실생활 활용, 학습자 주도성, 독서·작문 연계를 강조하고 있고, KICE 집필기준도 쓰기 제재를 읽기 경험과 연결해야 한다고 밝힙니다. 읽은 후 쓰기 과제 연구 역시 한국어 학술/에세이 쓰기에서 읽기 자료 처리 능력이 핵심임을 보여 줍니다. ([교육부][1])

또한 효과적인 쓰기 교육은 결과물 하나를 제출시키는 방식이 아니라, **계획하기·목표 세우기·초고 쓰기·평가하기·고쳐쓰기·편집하기**를 명시적으로 가르치는 과정 중심 수업이어야 합니다. IES의 쓰기 가이드와 NCTE 자료는 계획-초고-공유-수정-편집-평가-출판의 순환을 강조하고, 한국어 쓰기 연구도 브레인스토밍, 워크숍, 동료 피드백, 수정본 작성, 출판 단계를 핵심으로 제시합니다. ([교육과학연구소][2])

그리고 초심자에게 곧바로 장문 에세이를 요구하면 실패합니다. 최근 한국어 글쓰기 연구는 **단어→문장→문단→글**의 점진적 확장이 초보자의 부담을 낮추고 효능감을 높인다고 보며, 문단 구조 연구는 중심문장·뒷받침문장·도입부·종결부 이해가 부족하면 글 전체 구조가 무너진다고 지적합니다. 개요 작성 교육도 별도 활동으로 다뤄야 실제 글의 논리 흐름을 잡는 데 도움이 됩니다. ([DBpia][3])

따라서 플랫폼의 커리큘럼은 아래 8개 활동군으로 전부 나뉘어야 합니다.

## 1. 진단 활동군

사용자가 아무것도 모를 수도, 어느 정도 알 수도 있으므로 시작은 항상 진단이어야 합니다. IES는 전략 지도 강도를 학습자 수준에 맞게 조정해야 한다고 권고합니다. ([교육과학연구소][4])

필수 진단은 다음입니다.

1. 읽기 진단
   글의 핵심문장 찾기, 문단 요약, 주장/근거 구분, 글의 목적 파악

2. 표현 진단
   한 문장 쓰기, 세 문장 쓰기, 한 문단 쓰기, 3문단 글쓰기

3. 언어 기초 진단
   문장 완결성, 연결어, 어휘 선택, 문어체/구어체, 맞춤법/띄어쓰기

4. 자기조절 진단
   주제 고르기 어려움, 계획 세우기 어려움, 수정 회피, 피드백 수용 태도

진단 결과로 사용자를 네 트랙으로 자동 배치합니다.

- A트랙: 읽기 입문 필요
- B트랙: 짧은 쓰기부터 시작
- C트랙: 문단 쓰기부터 시작
- D트랙: 에세이/자료기반 글쓰기부터 시작

## 2. 읽기 활동군

에세이를 못 쓰는 많은 사용자는 실제로 “글을 쓰는 법”보다 먼저 “글을 읽는 법”을 모릅니다. 읽기와 쓰기 통합 지도 연구는 읽기-쓰기 상호작용, 쓰기를 위한 읽기, 읽기를 위한 쓰기를 모두 중요한 모형으로 봅니다. ([DBpia][5])

플랫폼에 반드시 있어야 할 읽기 활동은 다음과 같습니다.

1. 문장 읽기
   문장 경계 찾기, 핵심어 표시, 문장 의미 풀어 말하기

2. 문단 읽기
   중심문장 찾기, 보조문장 분류, 문단 한 줄 요약

3. 글 구조 읽기
   도입-전개-종결 구분, 설명/주장/비평 구분, 필자 목적 파악

4. 비판적 읽기
   주장 찾기, 근거의 강도 판단, 빠진 정보 찾기, 반론 상상하기

5. 읽기 후 반응
   공감/비판 한 문장 쓰기, 질문 1개 만들기, 내 경험 연결하기

6. 자료 읽기
   두 글 비교, 공통점/차이점 정리, 근거 카드 만들기

즉, 플랫폼의 첫 레벨은 “독해 강의”가 아니라 **에세이를 쓰기 위한 읽기 훈련**이어야 합니다.

## 3. 생각 생성 활동군

초보자는 할 말이 없어서 못 쓰는 경우가 많습니다. 그래서 쓰기 전 활동이 반드시 분리돼야 합니다. 한국어 쓰기 연구에서도 브레인스토밍, 다발짓기, 마인드맵 같은 전쓰기 활동이 초보자의 부담 감소와 효능감 향상에 유효하다고 제시됩니다. ([DBpia][3])

필수 활동은 다음입니다.

1. 관찰 기록
2. 경험 회상
3. 질문 만들기
4. 자유연상
5. 마인드맵
6. 주제 좁히기
7. 독자 상상하기
8. 글의 목적 정하기
9. 찬반 입장 정리
10. 근거 후보 수집

## 4. 짧은 쓰기 활동군

초심자에게는 “짧게 성공하는 경험”이 필요합니다. 그래서 에세이 이전에 반드시 짧은 쓰기 레벨이 있어야 합니다. ([DBpia][3])

필수 단위 활동은 다음과 같습니다.

1. 한 문장 요약
2. 한 문장 감상
3. 세 문장 요약
4. 세 문장 의견
5. 문장 완성
6. 틀 문장 바꿔 쓰기
7. 모방 문장 쓰기
8. 같은 뜻을 다른 표현으로 쓰기
9. 이유 한 문장 덧붙이기
10. 예시 한 문장 덧붙이기

이 단계의 목표는 “잘 쓰기”가 아니라 **완결된 문장을 두려움 없이 만드는 것**입니다.

## 5. 문단 쓰기 활동군

문단은 에세이의 최소 단위입니다. 문단 구조 연구와 문단 쓰기 지도 연구는 중심문장과 뒷받침문장 이해가 부족하면 이후 에세이 품질도 같이 무너진다고 보여 줍니다. ([DBpia][6])

필수 활동은 다음입니다.

1. 중심문장 쓰기
2. 중심문장과 무관한 문장 제거하기
3. 이유 2개 붙이기
4. 예시 1개 붙이기
5. 설명 문단 쓰기
6. 주장 문단 쓰기
7. 비교 문단 쓰기
8. 원인-결과 문단 쓰기
9. 전환어 넣기
10. 문단 고쳐쓰기

이 단계의 산출물은 “한 문단 자기 생각”, “한 문단 설명”, “한 문단 주장”입니다.

## 6. 에세이 구성 활동군

개요 작성과 구조화는 별도 학습이어야 합니다. 개요 작성 교육 연구는 이론 설명보다 실제 전략 체득이 중요하다고 말합니다. ([DBpia][7])

플랫폼에는 다음 활동이 있어야 합니다.

1. 주제문 만들기
2. 논지 한 문장 만들기
3. 독자와 목적 설정
4. 3문단 개요 작성
5. 5문단 개요 작성
6. 도입 쓰기
7. 본문 문단 배치
8. 결론 쓰기
9. 제목 붙이기
10. 역개요(reverse outline) 작성

이 단계부터 장르를 분화합니다.

- 감상 에세이
- 설명 에세이
- 주장 에세이
- 비평 에세이
- 자료기반 에세이

## 7. 자료기반 에세이 활동군

한국어 읽은 후 쓰기 연구는 실제 학술적 쓰기 과제가 읽기 자료 처리에 크게 의존한다고 봅니다. 따라서 중급 이상 사용자는 반드시 자료기반 글쓰기를 배워야 합니다. ([DBpia][8])

필수 활동은 다음입니다.

1. 자료에서 주장 찾기
2. 근거 카드 추출
3. 요약하기
4. 바꿔쓰기
5. 인용하기
6. 두 자료 비교
7. 내 입장 정리
8. 반론 넣기
9. 근거와 내 해설 연결
10. 참고 정보 정리

## 8. 피드백·고쳐쓰기 활동군

피드백과 수정은 선택이 아니라 핵심입니다. NCTE는 초고 코멘트, 동료 반응, 개별 컨퍼런스를 형성평가의 대표 방식으로 들고 있고, 한국어 쓰기 연구는 동료 피드백이 효과적이지만 사전 훈련이 필요하다고 봅니다. ([NCTE][9])

필수 활동은 다음입니다.

1. 자기 점검 체크리스트
2. 동료 피드백 읽는 법 배우기
3. 좋은 피드백/나쁜 피드백 구분하기
4. 내용 피드백
5. 구조 피드백
6. 문장 피드백
7. 피드백 수용 여부 결정
8. 수정 이유 쓰기
9. 2차 초고 작성
10. 최종 교정

중요한 점은, 피드백 화면이 단순 채점 화면이면 안 된다는 것입니다. 반드시
**문제 발견 → 수정 선택 → 수정 실행 → 수정 이유 기록**의 흐름이어야 합니다.

## 9. 출판·성찰 활동군

쓰기 워크숍 연구는 출판하기와 공유하기를 최종 단계로 둡니다. 이것이 학습자를 “답안 작성자”가 아니라 “필자”로 바꾸기 때문입니다. ([DBpia][10])

필수 활동은 다음입니다.

1. 낭독
2. 카드뉴스/블로그형 재구성
3. 모둠 공유
4. 포트폴리오 저장
5. 이번 글의 강점 1개 쓰기
6. 다음 글의 목표 1개 쓰기
7. 수정 전/후 비교
8. 장르별 대표작 선정

# 플랫폼 설계안: 활동 단위 구조

플랫폼의 최소 단위는 강의가 아니라 아래 형태의 **Activity Unit**이어야 합니다.

- 목표: 이번 활동에서 익힐 기술 1개
- 입력: 읽을 글, 질문, 예시
- 행동: 표시하기/고르기/말하기/쓰기/수정하기
- 산출물: 1문장, 3문장, 1문단, 개요, 초고 등
- 피드백: 자동 피드백 + 자기점검 + 동료/튜터 피드백
- 통과 조건: mastery 기준 충족 여부
- 다음 활동 연결: 부족한 하위 기술 보정 또는 상위 활동 진입

예를 들면 이런 식입니다.

**활동 예시 1: 한 문단 요약**

- 입력: 250자 내외 글
- 행동: 핵심문장 표시 → 핵심어 3개 선택 → 1문단 요약 작성
- 산출물: 80~120자 요약
- 피드백: 빠진 핵심/불필요한 세부/복붙 비율 안내
- 선행 조건: 문장 읽기 3회 통과

**활동 예시 2: 주장 문단 쓰기**

- 입력: 논쟁적 질문 1개
- 행동: 입장 선택 → 이유 2개 → 예시 1개 → 전환어 넣기
- 산출물: 150~250자 문단
- 피드백: 중심문장 존재 여부, 이유-예시 연결성
- 선행 조건: 중심문장 활동 통과

**활동 예시 3: 자료기반 짧은 에세이**

- 입력: 상반된 관점의 짧은 글 2편
- 행동: 각 글 요약 → 근거 카드 추출 → 내 논지 작성 → 3문단 에세이 초고
- 산출물: 500~700자 에세이
- 피드백: 자료 오독, 근거 부족, 반론 누락
- 선행 조건: 요약하기·개요 작성·문단 3종 통과

# 레벨 구조

## 레벨 0. 예비 문해/독해 입문

대상: 읽는 방법 자체가 약한 사용자
목표: 문장·문단을 읽고 핵심을 말할 수 있음

핵심 활동:

- 문장 의미 풀어 말하기
- 핵심어 표시
- 문단 요약
- 질문 만들기
- 필자 목적 찾기

## 레벨 1. 짧은 쓰기

대상: 문장은 읽지만 쓰기를 두려워하는 사용자
목표: 짧은 완결문을 안정적으로 씀

핵심 활동:

- 한 문장 반응
- 세 문장 요약
- 세 문장 의견
- 모방 쓰기
- 문장 고치기

## 레벨 2. 문단 쓰기

대상: 문장은 쓰지만 글 구조가 없는 사용자
목표: 중심문장-근거-예시를 갖춘 문단 작성

핵심 활동:

- 중심문장 쓰기
- 이유 붙이기
- 예시 붙이기
- 전환어 사용
- 문단 수정

## 레벨 3. 짧은 에세이

대상: 문단은 쓰지만 장문 구조가 약한 사용자
목표: 도입-본문-결론이 있는 3문단 글

핵심 활동:

- 논지 만들기
- 개요 작성
- 도입/결론 쓰기
- 초고 쓰기
- 자기 점검

## 레벨 4. 자료기반 에세이

대상: 자기 생각은 쓰지만 근거 사용이 약한 사용자
목표: 읽기 자료를 근거로 논지를 전개

핵심 활동:

- 자료 요약
- 근거 카드
- 비교/대조
- 반론 넣기
- 자료기반 수정

## 레벨 5. 심화 에세이

대상: 어느 정도 쓰지만 더 길고 복잡한 글이 필요한 사용자
목표: 비평문, 탐구형 에세이, 주제 확장 글 작성

핵심 활동:

- 다중 자료 통합
- 개념 정의
- 반론-재반박
- 문체 조정
- 포트폴리오 프로젝트

# 사용자 유형별 운영 방식

완전 초심자는
**읽기 입문 → 짧은 쓰기 → 문단 → 3문단 에세이 → 자료기반 에세이**
순으로 가야 합니다.

어느 정도 아는 사용자는
진단 후 부족한 활동만 보정하고 바로 상위 레벨로 올라가야 합니다. 예를 들어 문장은 잘 쓰지만 구조가 약하면 문단/개요 활동만 집중적으로 배정하면 됩니다. 이것이 플랫폼형 설계의 핵심입니다. ([교육과학연구소][4])

# 실제 서비스 화면 구조

플랫폼 정보구조는 아래처럼 두는 것이 좋습니다.

1. 진단 센터
2. 오늘의 활동
3. 읽기 워크벤치
4. 짧은 쓰기 연습장
5. 문단 스튜디오
6. 에세이 빌더
7. 피드백 워크숍
8. 수정 히스토리
9. 포트폴리오
10. 튜터/교사용 대시보드

특히 **에세이 빌더**는 완성형 편집기보다
“논지 카드 → 문단 카드 → 초고 → 수정본”
형태가 더 적합합니다. 초심자에게 빈 화면은 너무 어렵기 때문입니다.

# 평가 체계

평가는 최종 점수 하나로 끝내면 안 됩니다. 형성평가 중심이어야 합니다. ([NCTE][9])

평가 축은 7개면 충분합니다.

- 읽기 이해
- 주제/논지
- 근거 사용
- 조직
- 문장 명료성
- 어휘/문체
- 수정 능력

여기에 별도로 **성장 지표**를 둡니다.

- 초고 대비 수정률
- 피드백 반영률
- 문단 구조 안정도
- 자료 오독 감소율
- 자기 점검 정확도

# 최종 설계 원칙 한 줄 정리

이 플랫폼은 “에세이를 잘 써 주세요”라고 답을 대신 만들어 주는 서비스가 아니라,
**읽는 법을 배우고, 짧게 쓰며, 문단을 만들고, 개요를 세우고, 자료를 읽어 근거를 고르고, 초고를 쓰고, 피드백을 받고, 고쳐 쓰고, 출판하며, 점점 더 긴 글로 올라가는 시스템**이어야 합니다. 이 구조가 한국어 교육과정의 방향, 읽기-쓰기 통합 연구, 과정 중심 쓰기, 문단/개요 교육, 동료 피드백 연구와 가장 잘 맞습니다. ([교육부][1])
