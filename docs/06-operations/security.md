## 관점

글필의 보안은 계정 보안에 국한되지 않습니다. 사용자는 서재에 초안과 완성 글을 보관하며, AI 피드백을 통해 개인적 사고 과정을 시스템에 노출합니다. 글 내용, 여정 진행 데이터, AI 피드백 원문은 개인정보에 준하는 민감 데이터입니다.

## 위협 모델

- 계정 탈취와 세션 하이재킹 (Google/Kakao 소셜 로그인 포함)
- 권한 상승과 접근 제어 실패 (사용자 간 글/여정 데이터 교차 접근)
- 초안·완성 글 유출 (비공개 글 노출, API 응답 권한 오류)
- AI 피드백 내용 유출 (관측 시스템에 본문·피드백 내용 기록)
- CSRF 및 세션 고정 공격
- 결제 관련 부정 (Pro 구독 우회, 환불 악용)

## 우선순위

1. 계정과 세션 보호 (HttpOnly 쿠키, 세션 만료 관리)
2. 글·여정 데이터 접근 제어 (사용자 본인 데이터 격리)
3. AI 피드백 데이터 보호 (관측 시스템 복제 금지)
4. 감사 로그와 운영 추적성

## 기술 구현 원칙

- better-auth 세션: HttpOnly 쿠키, CSRF 미들웨어 최전단 적용
- 모든 API 엔드포인트: 인증 확인 + 소유권 검증 필수
- 글 본문·AI 피드백 내용: 로그·관측 시스템에 수집 금지 (→ [[../03-architecture/observability-architecture|관측 아키텍처]])
- 파일 저장소(R2): 서명 URL 단기 수명, 사용자 직접 업로드 금지 (→ [[../03-architecture/file-storage-strategy|파일 저장 전략]])

## 제품 반영 원칙

- 민감한 행위(글 삭제, 계정 탈퇴)는 확인 단계를 둔다.
- 운영자는 조치 근거를 감사 로그로 추적할 수 있어야 한다.
- 보안 정책은 운영 문서와 분리하지 않고 함께 설계한다.
