# Security

## 관점

이 플랫폼의 보안은 계정 보안만이 아니라, 비공개 글, 공개 범위, 커뮤니티 안전, 결제 신뢰까지 포함합니다.

## 위협 모델

- 계정 탈취와 세션 하이재킹
- 권한 상승과 접근 제어 실패
- 비공개 글 유출과 콘텐츠 변조
- 댓글, 메시지, 피드백 기반 괴롭힘과 스팸
- 결제 및 환불 부정

## 우선순위

1. 계정과 세션 보호
2. 글 공개 범위와 접근 제어
3. 신고와 차단을 포함한 커뮤니티 보호
4. 감사 로그와 운영 추적성

## 제품 반영 원칙

- 공개 범위는 명시적으로 보여야 한다.
- 민감한 행위는 확인 단계를 둔다.
- 운영자는 조치 근거를 추적할 수 있어야 한다.
- 보안 정책은 운영 문서와 분리하지 않고 함께 설계한다.
