## 목적

장애와 운영 사고가 발생했을 때 누가 무엇을 먼저 확인해야 하는지 빠르게 결정할 수 있어야 합니다.

## 사고 유형

- 서비스 접속 불가
- 저장 실패 또는 데이터 유실 의심
- 권한 오류로 인한 비공개 글 노출
- 결제 또는 정산 오류
- 악성 사용자 활동 급증

## 기본 대응

1. 영향 범위 파악
2. 신규 피해 확산 차단
3. 사용자가 체감하는 핵심 흐름부터 복구
4. 원인과 조치 내역 기록
5. 재발 방지 항목을 관련 문서에 반영

## 문서화 규칙

- 사고 후에는 운영 문서와 엔지니어링 문서를 함께 갱신합니다.
- 임시 우회책을 영구 정책처럼 남기지 않습니다.
