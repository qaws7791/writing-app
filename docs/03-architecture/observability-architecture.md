## 관측 목표

- 사용자가 글을 잃어버리는 상황을 가장 먼저 감지한다.
- 인증, 자동 저장, AI 보조, 공개 흐름의 병목을 구분해서 본다.
- 사용자 불편과 운영 원인을 같은 사건으로 연결해 추적할 수 있어야 한다.
- 제품 이벤트와 인프라 이벤트를 분리해 해석한다.

## 수집 계층

### 애플리케이션 로그

- `web`과 `api` 모두 구조화 로그를 남긴다.
- 모든 요청과 주요 도메인 이벤트에 `request_id`를 포함한다.
- 인증 관련 로그에는 사용자 식별자를 직접 남기지 않고 내부 식별자만 사용한다.
- 글 본문, AI 프롬프트 원문, 세션 토큰은 로그에 저장하지 않는다.

### 메트릭

- API 응답 시간, 오류율, 인증 실패율, 저장 성공률, AI 응답 시간, 파일 생성 실패율을 기본 메트릭으로 수집한다.
- 사용자 행동 메트릭과 운영 메트릭은 별도 네임스페이스로 관리한다.
- 자동 저장은 요청 수보다 성공적으로 커밋된 저장 수를 기준으로 본다.

### 분산 추적

- 브라우저 요청에서 시작한 추적을 API, 데이터베이스, 외부 AI 호출까지 연결한다.
- 느린 요청은 어느 계층에서 지연이 생겼는지 한 번에 확인할 수 있어야 한다.
- AI 요청은 모델 호출 시간과 후처리 시간을 분리해 기록한다.

### 감사 이벤트

- 로그인, 로그아웃, 세션 폐기, 공개 범위 변경, 글 삭제, 운영 제재 같은 행위는 감사 이벤트로 별도 저장한다.
- 감사 이벤트는 일반 로그보다 긴 보존 기간을 갖는다.

## 핵심 운영 지표

### 글쓰기 신뢰성

- 자동 저장 성공률
- 저장 지연 시간 p50, p95
- 충돌 또는 버전 생성 실패 수
- 에디터 진입 대비 첫 저장 완료 비율

### 인증과 세션

- 로그인 성공률
- 세션 만료 후 재인증 성공률
- CSRF 또는 권한 오류 비율
- 소셜 로그인 공급자별 실패율

### AI 보조

- 선택 영역 제안 응답 시간
- 문서 검토 응답 시간
- AI 요청 실패율
- 사용자가 제안을 적용한 비율

### 공개와 공유

- 공개 설정 변경 수
- 제한 공개 링크 생성 성공률
- 공개 후 되돌리기 비율

## 대시보드 원칙

- 제품 대시보드: 홈 진입, 글감 선택, 작성 시작, 저장, 공개까지의 퍼널
- 운영 대시보드: 오류율, 지연 시간, 외부 의존성 상태, 배포 직후 이상 징후
- 품질 대시보드: 브라우저 오류, 에디터 크래시, 장치별 성능 문제

## 알림 정책

- 저장 실패율 급증은 즉시 페이지 알림과 운영 알림을 함께 보낸다.
- 인증 실패 급증은 공급자 문제와 내부 문제를 구분해 알린다.
- AI 지연은 기능 비활성화 기준과 연계한다.
- 단발성 오류보다 일정 시간 지속되는 비정상 패턴에 우선 반응한다.

## 개인정보와 보존 정책

- 원문 글 내용은 관측 시스템에 복제하지 않는다.
- 로그는 개인정보 최소 수집 원칙을 따른다.
- 디버깅이 필요할 때도 본문 전문 대신 길이, 문단 수, 기능 사용 여부 같은 메타데이터를 우선 사용한다.
- 보존 기간은 로그, 메트릭, 트레이스, 감사 이벤트별로 다르게 설정한다.

## 장애 대응 연결

- 모든 사용자 노출 오류는 `request_id`를 포함해 고객 지원과 운영 대응을 연결한다.
- 배포 이벤트와 주요 경고를 같은 타임라인에서 볼 수 있어야 한다.
- 회고 문서에는 증상, 탐지 시간, 완화 시간, 재발 방지 조치를 함께 남긴다.
