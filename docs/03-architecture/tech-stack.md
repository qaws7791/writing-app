## `packages/db` — 데이터베이스

| 기술                                                        | 선택 이유                                                                                                                                                                                         |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[Drizzle ORM](https://orm.drizzle.team)**                 | 타입 안전한 SQL-first ORM. Prisma 대비 런타임 오버헤드가 작고, SQL에 가까운 직관적인 쿼리 API 제공. SQLite/PostgreSQL 마이그레이션이 간편하며 번들 크기가 작음                                    |
| **[bun:sqlite](https://bun.sh/docs/api/sqlite)** (WAL mode) | Bun 내장 SQLite 드라이버로 추가 의존성 없음. WAL(Write-Ahead Logging) 모드 활성화 시 읽기·쓰기 동시성이 크게 향상되며, 에세이 자동저장처럼 빈번한 쓰기 작업에 적합                                |
| **[Litestream](https://litestream.io)**                     | SQLite의 변경 사항을 WAL 페이지 단위로 S3/GCS 등 오브젝트 스토리지에 실시간 스트리밍 복제. 별도 DB 서버 없이 단일 인스턴스에서 재해 복구(DR)를 구현 가능. 하루 수 원 수준의 매우 저렴한 운영 비용 |

## `apps/api` — API 서버

### API Framework

| 기술                                                           | 선택 이유                                                                                                                                             |
| -------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[Hono](https://hono.dev)**                                   | Bun, Node.js, Cloudflare Workers 등 멀티 런타임 지원. 초경량 코어(< 14kb)로 엣지 환경까지 배포 가능. Express보다 빠르며 미들웨어 타입이 완전히 추론됨 |
| **[@hono/zod-openapi](https://hono.dev/snippets/zod-openapi)** | Zod 스키마로 라우트를 정의하면 OpenAPI 3.1 스펙을 자동 생성. 스키마·유효성 검사·문서화를 한 곳에서 관리하여 API와 문서의 동기화 문제를 원천 차단      |
| **[openapi-typescript](https://openapi-ts.dev)**               | 생성된 OpenAPI 스펙에서 TypeScript 타입을 자동 생성. `packages/db`의 DB 타입과 함께 프론트엔드까지 타입이 전파되는 E2E 타입 안전성 확보               |
| **[openapi-fetch](https://openapi-ts.dev/openapi-fetch/)**     | `openapi-typescript`로 생성된 타입을 기반으로 타입 안전한 fetch 클라이언트 생성. `apps/web`에서 API 호출 시 자동완성 및 타입 검사 지원                |

### API Documentation

| 기술                                                 | 선택 이유                                                                                                                           |
| ---------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| **[@scalar/hono-api-reference](https://scalar.com)** | Hono와 직접 통합되는 모던 API 문서 UI. Swagger UI보다 미려한 인터페이스와 Try It 기능 제공. `/docs` 엔드포인트 단 한 줄로 연결 가능 |

### 인증

| 기술                                       | 선택 이유                                                                                                                                                                                                                           |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[better-auth](https://better-auth.com)** | TypeScript 전용으로 설계된 풀스택 인증 프레임워크. 이메일/소셜 로그인, 2FA, 세션 관리, 패스키를 플러그인 형태로 조합 가능. Drizzle ORM 어댑터와 네이티브 통합으로 별도 auth 스키마 마이그레이션이 자동화됨. Hono 미들웨어 공식 지원 |

### 이메일

| 기술                             | 선택 이유                                                                                                                                                                   |
| -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[Resend](https://resend.com)** | better-auth의 이메일 발송(인증 메일·비밀번호 재설정)에 사용. 개발자 친화적 API, React 컴포넌트로 이메일 템플릿 작성 가능(`react-email` 통합). 무료 티어에서 월 3,000건 제공 |

### AI

| 기술                                         | 선택 이유                                                                                                                                                                                          |
| -------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[AI SDK (Vercel)](https://sdk.vercel.ai)** | OpenAI API를 비롯한 다양한 LLM 프로바이더를 통일된 인터페이스로 추상화. 스트리밍 응답, 구조화된 출력(structured output)을 간단하게 구현 가능. 기획서의 "사고 훈련 목적 AI 피드백" 기능 구현에 최적 |

## `apps/web` — 웹 프론트엔드

### Framework & Styling

| 기술                                           | 선택 이유                                                                                                                                                     |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[Next.js](https://nextjs.org)** (App Router) | React 기반 풀스택 프레임워크. App Router의 서버 컴포넌트로 초기 로딩 성능 최적화. 모바일 우선(max-width 600px) 에디터 화면에서 빠른 FCP/LCP 달성 가능         |
| **[React](https://react.dev)**                 | 컴포넌트 기반 UI 개발의 사실상 표준. `packages/ui`와 `apps/web` 간 컴포넌트 공유의 기반                                                                       |
| **[Tailwind CSS](https://tailwindcss.com)**    | 유틸리티 퍼스트 CSS. 디자인 토큰을 `tailwind.config`에서 중앙 관리하여 에디토리얼 감성의 일관된 타이포그래피·색상 시스템 유지. `packages/ui`와 공유 설정 가능 |

### 에디터

| 기술                             | 선택 이유                                                                                                                                                                                 |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[Tiptap](https://tiptap.dev)** | ProseMirror 기반의 헤드리스 리치 텍스트 에디터. 익스텐션 아키텍처로 자동저장, 버전 스냅샷, 하이라이트, 글자 수 카운터 등 기획서의 에세이 친화 UX를 커스텀 구현 가능. Tailwind와 완전 호환 |

### 상태 관리

| 기술                                        | 선택 이유                                                                                                                                                      |
| ------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[Zustand](https://zustand-demo.pmnd.rs)** | 최소한의 보일러플레이트로 클라이언트 전역 상태 관리. 에디터 초고/퇴고 모드 전환, 오프라인 임시저장 큐 등의 로컬 상태에 적합. Next.js 서버 컴포넌트와 충돌 없음 |

### 오프라인 지원

| 기술                                                                      | 선택 이유                                                                                                                                         |
| ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[Workbox](https://developer.chrome.com/docs/workbox)** (via `next-pwa`) | 기획서의 "오프라인 자동저장 + 재연결 시 동기화" 요구사항 구현. Service Worker 기반으로 네트워크 불안정 환경(모바일)에서도 에디터 데이터 유실 방지 |

## `packages/ui` — UI 컴포넌트

| 기술                                            | 선택 이유                                                                                                                                                                                                                 |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[React](https://react.dev)**                  | `apps/web`과 동일한 런타임. 컴포넌트를 `packages/ui`에서 정의하고 `apps/web`에서 그대로 import하는 공유 전략의 기반                                                                                                       |
| **[Base UI (Radix 후속)](https://base-ui.com)** | MUI 팀이 개발한 완전 비스타일 헤드리스 컴포넌트 라이브러리. 접근성(WCAG 2.2) 처리가 내장되어 있으며 CSS를 일절 번들하지 않아 Tailwind와 충돌 없음. Dialog, Select, Tooltip 등 복잡한 인터랙션을 처음부터 구현할 필요 없음 |
| **[shadcn/ui](https://ui.shadcn.com)**          | Base UI 위에 Tailwind로 스타일링된 고품질 컴포넌트 레시피. 코드를 직접 복사·소유하는 방식으로 `packages/ui`에 완전히 통합 가능. 빠른 프로토타이핑과 커스터마이징을 동시에 달성                                            |
| **[Tailwind CSS](https://tailwindcss.com)**     | `apps/web`과 동일한 Tailwind 설정을 `packages/ui`에서 공유. 디자인 시스템 토큰(폰트, 색상, 간격)을 단일 소스로 관리                                                                                                       |
