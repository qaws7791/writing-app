# 여정 스탭 설계

> 100개 여정 초기 데이터 분석을 기반으로 세션 내 스탭(Step)의 유형을 정의하고, 수평 확장이 용이한 공용 JSON 스키마를 설계한다.

---

## 1. 개요

### 1.1 계층 구조

```
Journey (여정)
 └── JourneySession (세션) — 10~20분 완결형 학습 단위
      └── Step (스탭) — 마이크로 활동 단위
```

### 1.2 세션 내 스탭 규칙

| 규칙         | 설명                                                          |
| ------------ | ------------------------------------------------------------- |
| 첫 번째 스탭 | 항상 `INTRO` — 해당 세션의 목적과 흐름을 소개                 |
| 마지막 스탭  | 항상 `COMPLETION` — 축하 메시지와 세션 결과 정리              |
| 진행 방식    | 모든 스탭은 하단 **CTA 버튼**을 통해 다음으로 이동            |
| 저장 분리    | 스탭 콘텐츠, 사용자 응답, AI 피드백은 각각 독립 엔티티로 저장 |
| 순서         | 세션 내 선형 순서(`order`)로 배치                             |

### 1.3 설계 원칙

1. **공용 스키마 기반 확장**: 모든 스탭은 동일한 `BaseStep` 스키마를 공유하고, `type` 필드로 분기하는 Discriminated Union 구조
2. **데이터 분리**: 스탭 정의(콘텐츠) / 사용자 응답(Response) / AI 피드백(Feedback) 3계층 분리
3. **CTA 중심 진행**: 자동 전환 없이, 사용자가 CTA를 눌러야 다음 스탭으로 이동
4. **수평 확장**: 새 스탭 유형 추가 시 `StepType`에 리터럴 추가 + `content` 스키마 정의만으로 확장

## 2. 스탭 계열 분류

5개 계열, 13개 스탭 유형으로 분류한다.

```
┌─────────────────────────────────────────────────────────┐
│                    Step Type Family                     │
├────────────┬────────────────────────────────────────────┤
│ 안내 계열   │ INTRO · COMPLETION                        │
│ (Guide)    │                                            │
├────────────┼────────────────────────────────────────────┤
│ 콘텐츠 계열 │ CONCEPT · EXAMPLE                          │
│ (Content)  │                                            │
├────────────┼────────────────────────────────────────────┤
│ 선택 계열   │ MULTIPLE_CHOICE · FILL_IN_THE_BLANK ·     │
│ (Selection)│ ORDERING · HIGHLIGHT                       │
├────────────┼────────────────────────────────────────────┤
│ 입력 계열   │ SHORT_ANSWER · WRITING · REWRITING         │
│ (Input)    │                                            │
├────────────┼────────────────────────────────────────────┤
│ 피드백 계열 │ AI_FEEDBACK · AI_COMPARISON                │
│ (Feedback) │                                            │
└────────────┴────────────────────────────────────────────┘
```

### 기존 PRD 유형 매핑

| PRD 원본            | 새 유형명           | 변경 사유                         |
| ------------------- | ------------------- | --------------------------------- |
| `LEARN`             | `CONCEPT`           | 개념/이론 설명임을 명확히         |
| `READ`              | `EXAMPLE`           | 예시문/모범글 읽기임을 명확히     |
| `GUIDED_QUESTION`   | `SHORT_ANSWER`      | 입력 계열 통일 (짧은 텍스트 입력) |
| `WRITE`             | `WRITING`           | 입력 계열 통일                    |
| `FEEDBACK`          | `AI_FEEDBACK`       | 피드백 계열 분리, AI 주체 명시    |
| `REVISE`            | `REWRITING`         | 입력 계열 통일, 수정 행위 명확화  |
| `FILL_IN_THE_BLANK` | `FILL_IN_THE_BLANK` | 유지                              |
| _(신규)_            | `INTRO`             | 세션 도입 (필수)                  |
| _(신규)_            | `COMPLETION`        | 세션 마무리 (필수)                |
| _(신규)_            | `MULTIPLE_CHOICE`   | 객관식 선택                       |
| _(신규)_            | `ORDERING`          | 순서 배열                         |
| _(신규)_            | `HIGHLIGHT`         | 텍스트 선택/분석                  |
| _(신규)_            | `AI_COMPARISON`     | 초안-수정본 비교 분석             |

## 3. 공용 기본 스키마

### 3.1 BaseStep (스탭 정의)

```typescript
/** 스탭 — 세션 내 마이크로 활동 단위 */
interface Step {
  /** 스탭 고유 ID */
  id: StepId
  /** 소속 세션 ID */
  sessionId: SessionId
  /** 스탭 유형 (Discriminator) */
  type: StepType
  /** 세션 내 순서 (1-based) */
  order: number
  /** 유형별 콘텐츠 (JSON) */
  content: StepContent
  /** CTA 버튼 설정 */
  cta: CTAConfig
}

/** 스탭 유형 열거 */
type StepType =
  // 안내 계열
  | "INTRO"
  | "COMPLETION"
  // 콘텐츠 계열
  | "CONCEPT"
  | "EXAMPLE"
  // 선택 계열
  | "MULTIPLE_CHOICE"
  | "FILL_IN_THE_BLANK"
  | "ORDERING"
  | "HIGHLIGHT"
  // 입력 계열
  | "SHORT_ANSWER"
  | "WRITING"
  | "REWRITING"
  // 피드백 계열
  | "AI_FEEDBACK"
  | "AI_COMPARISON"

/** CTA 버튼 설정 */
interface CTAConfig {
  /** 버튼 텍스트 */
  label: string
  /** 버튼 스타일 */
  variant: "primary" | "secondary"
}

/** 유형별 콘텐츠 — Discriminated Union */
type StepContent =
  | IntroContent
  | CompletionContent
  | ConceptContent
  | ExampleContent
  | MultipleChoiceContent
  | FillInTheBlankContent
  | OrderingContent
  | HighlightContent
  | ShortAnswerContent
  | WritingContent
  | RewritingContent
  | AIFeedbackContent
  | AIComparisonContent
```

### 3.2 StepResponse (사용자 응답 — 별도 저장)

```typescript
/** 사용자가 스탭에 제출한 응답 */
interface StepResponse {
  id: StepResponseId
  stepId: StepId
  userId: UserId
  sessionProgressId: SessionProgressId
  /** 응답 유형 (스탭 유형과 1:1 매핑) */
  type: StepResponseType
  /** 유형별 응답 데이터 (JSON) */
  data: StepResponseData
  submittedAt: Date
}

type StepResponseType =
  | "MULTIPLE_CHOICE_ANSWER"
  | "FILL_IN_THE_BLANK_ANSWER"
  | "ORDERING_ANSWER"
  | "HIGHLIGHT_ANSWER"
  | "SHORT_ANSWER_TEXT"
  | "WRITING_TEXT"
  | "REWRITING_TEXT"
```

### 3.3 StepFeedback (AI 피드백 — 별도 저장)

```typescript
/** AI가 생성한 스탭 피드백 */
interface StepFeedback {
  id: StepFeedbackId
  stepId: StepId
  responseId: StepResponseId
  userId: UserId
  /** 피드백 유형 */
  type: StepFeedbackType
  /** 피드백 데이터 (JSON) */
  data: StepFeedbackData
  generatedAt: Date
}

type StepFeedbackType =
  | "COACHING" // 강점/개선점/질문 피드백
  | "COMPARISON" // 초안-수정본 비교 분석
```

## 4. 스탭 유형별 상세 정의

### 4.1 안내 계열 (Guide)

#### INTRO — 세션 소개

> 세션의 목적, 학습 목표, 진행 흐름을 안내한다. **모든 세션의 첫 번째 스탭.**

```typescript
interface IntroContent {
  type: "INTRO"
  /** 세션 제목 */
  title: string
  /** 세션 목표 설명 */
  description: string
  /** 이번 세션에서 다룰 핵심 키워드 (선택) */
  keywords?: string[]
  /** 예상 소요 시간 (분) */
  estimatedMinutes: number
}
```

| 항목        | 설명                                        |
| ----------- | ------------------------------------------- |
| 사용자 응답 | 없음                                        |
| CTA 기본값  | `{ label: "시작하기", variant: "primary" }` |
| 사용 빈도   | 모든 세션 (필수)                            |

---

#### COMPLETION — 세션 완료

> 세션에서 수행한 활동을 요약하고 축하한다. **모든 세션의 마지막 스탭.**

```typescript
interface CompletionContent {
  type: "COMPLETION"
  /** 축하 메시지 */
  congratsMessage: string
  /** 세션 요약 포인트 */
  summaryPoints: string[]
  /** 다음 세션 미리보기 (마지막 세션이 아닌 경우) */
  nextSessionPreview?: {
    title: string
    teaser: string
  }
}
```

| 항목        | 설명                                                                               |
| ----------- | ---------------------------------------------------------------------------------- |
| 사용자 응답 | 없음                                                                               |
| 동적 데이터 | 서버가 해당 세션의 사용자 응답 통계를 집계하여 결과 카드에 병합                    |
| CTA 기본값  | `{ label: "다음 세션으로", variant: "primary" }` (마지막 세션이면 `"여정 마무리"`) |
| 사용 빈도   | 모든 세션 (필수)                                                                   |

---

### 4.2 콘텐츠 계열 (Content)

#### CONCEPT — 개념 학습

> 핵심 개념, 이론, 원칙을 설명한다. 사용자는 읽고 넘어간다.

```typescript
interface ConceptContent {
  type: "CONCEPT"
  /** 소제목 */
  title: string
  /** 본문 (Markdown) */
  body: string
  /** 핵심 요약 (하이라이트 박스) */
  keyTakeaway?: string
  /** 참고 이미지 URL (선택) */
  imageUrl?: string
}
```

| 항목        | 설명                                                                             |
| ----------- | -------------------------------------------------------------------------------- |
| 사용자 응답 | 없음                                                                             |
| CTA 기본값  | `{ label: "다음", variant: "primary" }`                                          |
| 사용 예시   | "에세이의 3단 구조", "귀납과 연역의 차이", "논리적 오류 유형", "역피라미드 구조" |

---

#### EXAMPLE — 예시문 읽기

> 모범 에세이, 문단, 문장 예시를 보여준다. 좋은 글/나쁜 글 비교도 가능하다.

```typescript
interface ExampleContent {
  type: "EXAMPLE"
  /** 소제목 */
  title: string
  /** 예시문 목록 */
  examples: Array<{
    /** 예시 레이블 (예: "좋은 예", "개선 전") */
    label?: string
    /** 예시 본문 */
    text: string
    /** 강조할 부분 (줄/구간 하이라이트) */
    highlights?: Array<{
      startOffset: number
      endOffset: number
      comment: string
    }>
  }>
  /** 예시에 대한 해설 */
  commentary?: string
}
```

| 항목        | 설명                                                                           |
| ----------- | ------------------------------------------------------------------------------ |
| 사용자 응답 | 없음                                                                           |
| CTA 기본값  | `{ label: "다음", variant: "primary" }`                                        |
| 사용 예시   | "오감을 활용한 묘사 예시", "좋은 이메일 vs 나쁜 이메일", "효과적인 오프닝 3선" |

### 4.3 선택 계열 (Selection)

#### MULTIPLE_CHOICE — 객관식 선택

> 주어진 선택지 중 정답 또는 적절한 항목을 고른다.

```typescript
interface MultipleChoiceContent {
  type: "MULTIPLE_CHOICE"
  /** 질문 */
  question: string
  /** 설명/지문 (선택) */
  passage?: string
  /** 선택지 목록 */
  options: Array<{
    id: string
    text: string
  }>
  /** 정답 ID 목록 (복수 정답 가능) */
  correctOptionIds: string[]
  /** 복수 선택 허용 여부 */
  multiSelect: boolean
  /** 정답/오답 해설 */
  explanations: Record<string, string>
}
```

**사용자 응답:**

```typescript
interface MultipleChoiceAnswer {
  type: "MULTIPLE_CHOICE_ANSWER"
  selectedOptionIds: string[]
  isCorrect: boolean
}
```

| 항목       | 설명                                                                     |
| ---------- | ------------------------------------------------------------------------ |
| CTA 기본값 | `{ label: "확인하기", variant: "primary" }`                              |
| 동작       | 선택지 탭 → CTA 클릭 → 정답 여부 및 해설 표시 → CTA가 "다음"으로 변경    |
| 사용 예시  | "다음 중 논리적 오류에 해당하는 것은?", "사실 진술과 의견 진술 구분하기" |

---

#### FILL_IN_THE_BLANK — 빈칸 채우기

> 문장 내 빈칸에 적절한 단어/구절을 선택지에서 골라 채운다.

```typescript
interface FillInTheBlankContent {
  type: "FILL_IN_THE_BLANK"
  /** 안내 문구 */
  instruction: string
  /** 빈칸이 포함된 문장 (빈칸은 `{{blank_id}}` 로 표시) */
  sentence: string
  /** 빈칸별 선택지 */
  blanks: Array<{
    id: string
    options: Array<{
      id: string
      text: string
    }>
    correctOptionId: string
  }>
  /** 해설 */
  explanation: string
}
```

**사용자 응답:**

```typescript
interface FillInTheBlankAnswer {
  type: "FILL_IN_THE_BLANK_ANSWER"
  selections: Array<{
    blankId: string
    selectedOptionId: string
  }>
  isCorrect: boolean
}
```

| 항목       | 설명                                                                       |
| ---------- | -------------------------------------------------------------------------- |
| CTA 기본값 | `{ label: "완성 확인", variant: "primary" }`                               |
| 동작       | 각 빈칸의 보기 중 선택 → CTA 클릭 → 정답 여부 표시 → CTA가 "다음"으로 변경 |
| 사용 예시  | "적절한 전환어 넣기", "비유 표현 완성하기", "어휘 정확성 훈련"             |

---

#### ORDERING — 순서 배열

> 무작위로 섞인 항목들을 올바른 순서로 드래그하여 배열한다.

```typescript
interface OrderingContent {
  type: "ORDERING"
  /** 안내 문구 */
  instruction: string
  /** 배열할 항목 목록 (셔플되어 표시) */
  items: Array<{
    id: string
    text: string
  }>
  /** 정답 순서 (item ID 배열) */
  correctOrder: string[]
  /** 해설 */
  explanation: string
}
```

**사용자 응답:**

```typescript
interface OrderingAnswer {
  type: "ORDERING_ANSWER"
  submittedOrder: string[]
  isCorrect: boolean
}
```

| 항목       | 설명                                                                             |
| ---------- | -------------------------------------------------------------------------------- |
| CTA 기본값 | `{ label: "순서 확인", variant: "primary" }`                                     |
| 동작       | 항목 드래그/탭으로 순서 변경 → CTA 클릭 → 정답 비교 표시 → CTA가 "다음"으로 변경 |
| 사용 예시  | "에세이 단락 순서 맞추기", "논증 구조 배열", "논술 개요 구성요소 정렬"           |

---

#### HIGHLIGHT — 텍스트 선택

> 주어진 지문에서 특정 조건에 맞는 부분을 선택(하이라이트)한다.

```typescript
interface HighlightContent {
  type: "HIGHLIGHT"
  /** 안내 문구 (예: "아래 글에서 논리적 오류가 있는 부분을 선택하세요") */
  instruction: string
  /** 분석 대상 지문 */
  passage: string
  /** 선택 가능한 영역 (미리 정의된 구간) */
  selectableRanges: Array<{
    id: string
    startOffset: number
    endOffset: number
  }>
  /** 정답 영역 ID 목록 */
  correctRangeIds: string[]
  /** 영역별 해설 */
  explanations: Record<string, string>
}
```

**사용자 응답:**

```typescript
interface HighlightAnswer {
  type: "HIGHLIGHT_ANSWER"
  selectedRangeIds: string[]
  isCorrect: boolean
}
```

| 항목       | 설명                                                                               |
| ---------- | ---------------------------------------------------------------------------------- |
| CTA 기본값 | `{ label: "확인하기", variant: "primary" }`                                        |
| 동작       | 지문의 선택 가능 영역 탭 → CTA 클릭 → 정답/오답 하이라이트 → CTA가 "다음"으로 변경 |
| 사용 예시  | "글에서 핵심 논지 찾기", "논리적 오류 부분 선택", "제시문 핵심 논점 파악"          |

### 4.4 입력 계열 (Input)

#### SHORT_ANSWER — 짧은 답변

> 사고를 촉발하는 질문에 짧은 텍스트로 답변한다. 정답 없이 기록 목적.

```typescript
interface ShortAnswerContent {
  type: "SHORT_ANSWER"
  /** 질문 */
  question: string
  /** 질문 배경 설명 (선택) */
  context?: string
  /** 입력 힌트 (placeholder) */
  placeholder?: string
  /** 최소 글자 수 (0이면 제한 없음) */
  minLength: number
  /** 최대 글자 수 */
  maxLength: number
}
```

**사용자 응답:**

```typescript
interface ShortAnswerText {
  type: "SHORT_ANSWER_TEXT"
  text: string
}
```

| 항목       | 설명                                                                                                |
| ---------- | --------------------------------------------------------------------------------------------------- |
| CTA 기본값 | `{ label: "답변 제출", variant: "primary" }`                                                        |
| 동작       | 텍스트 입력 → CTA 클릭 → 응답 저장 → 다음 스탭 이동                                                 |
| 사용 예시  | "나를 정의하는 문장 쓰기", "이 예시의 핵심 논지는?", "오늘 쓸 소재 3가지", "이 주장에 대한 반론은?" |

---

#### WRITING — 글쓰기

> 본격적으로 에세이/문단/글을 작성한다. 자동 저장 + AI 분석 연계.

```typescript
interface WritingContent {
  type: "WRITING"
  /** 글쓰기 프롬프트 (주제/지시문) */
  prompt: string
  /** 프롬프트 보조 설명 */
  guideline?: string
  /** 참고용 글감 ID (선택 — WritingPrompt 연결) */
  writingPromptId?: WritingPromptId
  /** 최소 글자 수 */
  minLength: number
  /** 권장 글자 수 */
  recommendedLength: number
```

## 5. 세션 구성 패턴

### 5.1 일반 구성 흐름

모든 세션은 다음 기본 구조를 따른다:

```
INTRO → [학습/활동 스탭 N개] → COMPLETION
```

### 5.2 대표 세션 패턴

카테고리별 자주 사용되는 세션 패턴을 정리한다.

#### 패턴 A: 개념 학습 + 퀴즈

```
INTRO → CONCEPT → EXAMPLE → MULTIPLE_CHOICE → CONCEPT → FILL_IN_THE_BLANK → COMPLETION
```

적용: 에세이 구조와 논리, 비판적 사고, 입시와 학술 글쓰기

#### 패턴 B: 개념 학습 + 글쓰기 + AI 피드백

```
INTRO → CONCEPT → EXAMPLE → WRITING → AI_FEEDBACK → REWRITING → AI_COMPARISON → COMPLETION
```

적용: 글쓰기 기초, 감각과 묘사, 직무와 실용 글쓰기, 퍼블리싱

#### 패턴 C: 자기 탐구 (질문 중심)

```
INTRO → CONCEPT → SHORT_ANSWER → SHORT_ANSWER → WRITING → AI_FEEDBACK → COMPLETION
```

적용: 자기 탐구와 회고, 마음챙김과 감정 글쓰기

#### 패턴 D: 분석 + 비평

```
INTRO → CONCEPT → EXAMPLE → HIGHLIGHT → SHORT_ANSWER → WRITING → AI_FEEDBACK → COMPLETION
```

적용: 비판적 사고와 논증, 입시(제시문 분석)

#### 패턴 E: 구조화 훈련

```
INTRO → CONCEPT → ORDERING → EXAMPLE → FILL_IN_THE_BLANK → WRITING → AI_FEEDBACK → COMPLETION
```

적용: 에세이 구조와 논리, 창작과 스토리텔링

#### 패턴 F: 시간 제한 실전

```
INTRO → CONCEPT → WRITING(timeLimitSeconds > 0) → AI_FEEDBACK → COMPLETION
```

적용: 5분 글쓰기 습관, 시간 안에 완성하는 논술, 개요 30분 작성

## 6. CTA 버튼 규칙

### 6.1 공통 규칙

| 규칙      | 설명                                                        |
| --------- | ----------------------------------------------------------- |
| 위치      | 화면 하단 고정 (키보드 위)                                  |
| 활성 조건 | 스탭 유형별 필수 조건 충족 시 활성화                        |
| 단일 CTA  | 기본적으로 1개의 Primary CTA                                |
| 예외      | `AI_FEEDBACK`만 Primary("수정하기") + Secondary("다음") 2개 |

### 6.2 유형별 CTA 활성화 조건

| 스탭 유형           | 활성화 조건                       | 기본 라벨                                 |
| ------------------- | --------------------------------- | ----------------------------------------- |
| `INTRO`             | 즉시 활성                         | "시작하기"                                |
| `COMPLETION`        | 즉시 활성                         | "다음 세션으로"                           |
| `CONCEPT`           | 즉시 활성                         | "다음"                                    |
| `EXAMPLE`           | 즉시 활성                         | "다음"                                    |
| `MULTIPLE_CHOICE`   | 1개 이상 선택 시                  | "확인하기" → 확인 후 "다음"               |
| `FILL_IN_THE_BLANK` | 모든 빈칸 선택 시                 | "완성 확인" → 확인 후 "다음"              |
| `ORDERING`          | 순서 변경 시                      | "순서 확인" → 확인 후 "다음"              |
| `HIGHLIGHT`         | 1개 이상 선택 시                  | "확인하기" → 확인 후 "다음"               |
| `SHORT_ANSWER`      | 텍스트 입력 시 (`minLength` 충족) | "답변 제출"                               |
| `WRITING`           | 텍스트 입력 시 (`minLength` 충족) | "제출"                                    |
| `REWRITING`         | 텍스트 수정 시                    | "재제출"                                  |
| `AI_FEEDBACK`       | 피드백 로딩 완료 시               | "수정하기" (Primary) / "다음" (Secondary) |
| `AI_COMPARISON`     | 비교 분석 로딩 완료 시            | "다음"                                    |

### 6.3 2단계 CTA (선택 계열)

선택 계열 스탭(`MULTIPLE_CHOICE`, `FILL_IN_THE_BLANK`, `ORDERING`, `HIGHLIGHT`)은 2단계 CTA를 사용한다:

```
[1단계] 사용자 선택 → CTA("확인하기") 클릭 → 정답/오답 결과 표시
[2단계] 결과 확인 → CTA("다음") 으로 변경 → 클릭 → 다음 스탭 이동
```
