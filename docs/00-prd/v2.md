## 1. Executive Summary

**글필(Geulpil)**은 한국어 사용자를 위한 **여정 기반 글쓰기 교육 플랫폼**이다.

글필의 단 하나의 핵심 존재 이유는 **"여정(Journey)"**이다. 여정은 특정 글쓰기 역량을 목표로 설계된 구조화된 커리큘럼이며, 사용자는 여정을 통해 단계별로 사고력과 글쓰기 능력을 성장시킨다. 글필의 모든 기능, 모든 UX 결정, 모든 개발 우선순위는 여정 경험을 중심으로 수렴한다.

**글쓰기(Writing)**는 여정에서 배운 개념과 기술을 부담 없이 자유롭게 연습하는 개인 공간이다. 글쓰기는 여정의 연장선상에 위치하며, 독립적인 핵심 기능이 아닌 여정 학습의 자연스러운 출구이다.

**글감(Writing Prompt)**은 글쓰기 공간 내부의 보조 도구다. 빈 화면 앞에서 막막한 사용자에게 아이디어를 제공하는 역할만을 수행한다. 글감은 글쓰기를 위한 선택적 출발점이지, 그 자체가 목적이 아니다. 단, v2.1에서는 글쓰기 탭 진입 시 글감 카드를 미리 노출하는 중간 화면을 도입하여 발견성을 보강한다. 이는 글감의 Tertiary 위계를 유지하면서 독립 탭 없이 발견성을 확보하는 절충점이다.

이 위계 구조는 플랫폼의 설계 원칙인 **"Do One Thing Well"**을 구현한다. 글필은 여정을 통한 한국어 글쓰기 교육이라는 단 하나의 일을 탁월하게 수행하는 데 집중한다.

## 2. 문제 정의

### 2.1. 시대적 맥락

숏폼 콘텐츠와 알고리즘 기반 소비 문화는 주의 집중 시간을 단축하고, 비판적 사고보다 감정 반응을 우선하는 미디어 소비 습관을 형성했다. 글쓰기의 사회적 필요가 감소하면서(편지, 일기, 보고서 → 메신저, 이모지, 음성 메시지), 논리적 구성 능력과 표현력이 전반적으로 저하되고 있다. 생성형 AI 도구의 남용은 "모름 → 질문 → AI가 즉답"이라는 단축 경로를 제공하여, 스스로 사유하는 과정 자체를 건너뛰게 만든다.

### 2.2. 사용자가 겪는 구체적 고통

| 고통 영역           | 핵심 문제                                                  | 결과                     |
| ------------------- | ---------------------------------------------------------- | ------------------------ |
| **아이디어 생성**   | 무엇을 써야 할지 막막함, 소재/주제 생성 어려움             | 글쓰기 시작 자체를 회피  |
| **구성·구조화**     | 서론-본론-결론의 전체 틀과 문단 구성에 대한 불안           | 비구조적이고 산만한 글   |
| **완벽주의·미루기** | 초고를 "형편없다"고 판단하여 반복 갈아엎음                 | 품질/만족도 저하, 미완성 |
| **퇴고 어려움**     | 초고-퇴고-버전 관리가 어렵고 "나중에 고치기"가 무한 대기열 | 성장 없는 반복           |
| **학습 결핍**       | 에세이 구조, 퇴고 전략에 대한 체계적 학습 기회 부재        | 실력 정체                |
| **공개 불안**       | "내 글이 틀릴까", 타인의 평가에 대한 자기검열              | 글쓰기 중단, 침묵 효과   |
| **AI 의존**         | AI 생성 텍스트의 획일화, 개성 약화, 사고력 퇴화            | 비판적 사고 근육 상실    |

### 2.3. 기존 도구의 한계

**생산성 앱의 한계:** 정보를 수집·분류하는 행위 자체를 "사고했다"고 착각하게 만드는 수집가의 오류(Collector's Fallacy). 정보의 단순 축적은 통찰로 이어지지 않는다.

**모바일 앱의 한계:** 한 번에 보이는 정보량이 적어 전체 맥락 파악이 어렵고, 앱 알림과 메신저로 인해 글쓰기 흐름이 자주 끊긴다. 비선형적 사고를 지원하는 도구가 부족하다.

**생성형 AI의 한계:** 허위·환각 정보 생산, 문체 획일화, 저자 개성 약화, 과도한 의존에 의한 스킬 저하, "AI 티 나는 글"에 대한 거부감 등 근본적인 문제가 존재한다.

## 4. 레퍼런스 서비스 분석

### 4.1. Quill.org — 즉각적 AI 피드백 + 문장 단위 훈련

Quill은 1,000개 이상의 차별화된 활동을 제공하며, 학생이 문장을 제출하면 AI가 즉시 피드백을 제공한다. 핵심 도구는 6가지(Diagnostics, Connect, Grammar, Proofreader, Lessons, Reading for Evidence)이며, 활동당 10~15분의 짧은 세션 구조를 채택한다. 기술적 문법 용어 대신 기능 중심 설명을 사용하고, 수정을 글쓰기의 자연스러운 과정으로 격려한다.

**글필에의 시사점:** 짧은 세션 구조, 즉각적 AI 피드백, "정답/오답"이 아닌 코칭형 피드백 톤, 진단 → 맞춤 학습 계획 생성 구조를 여정 설계에 반영한다.

### 4.2. NoRedInk — 관심사 기반 개인화 + 적응형 학습

NoRedInk는 학생의 관심사를 활용하여 연습 문장을 개인화한다. AI 채점 보조로 학생에게 더 많은 피드백을 제공하며, 미국 학교의 60% 이상이 사용 중이다.

**글필에의 시사점:** 온보딩 시 사용자 관심사 수집 → 여정 및 글감 개인화, 적응형 난이도 조절, 관심사 기반 몰입감을 차용한다.

### 4.3. Write & Improve with Cambridge — 제출-피드백-수정 루프

Write & Improve는 CEFR 기준으로 글쓰기 수준을 평가하고, 세 가지 유형의 피드백을 제공한다. 90% 이상 확신할 때만 오류를 지적하며, 수정 후 재제출하면 새로운 피드백이 제공되는 반복 루프가 핵심이다.

**글필에의 시사점:** 수준별 자동 평가, "제출 → 피드백 → 수정 → 재제출" 루프, 점진적 피드백 노출, 격려 중심 톤을 여정 세션 설계의 근간으로 삼는다.

## 5. 타겟 사용자

### 5.1. 주요 페르소나

**페르소나 A — "글쓰기 입문자" 민지 (22세, 대학생)**

- 에세이 과제에서 늘 막막함을 느끼고, 구조를 잡는 방법을 모른다.
- AI에 의존해 과제를 제출하지만, 실력이 늘지 않는 것을 인지하고 있다.
- 원하는 것: 단계별로 글 쓰는 법을 배우고, 내 실력이 올라가는 것을 확인하고 싶다.
- 핵심 동기: 학업 성취, 취업 준비 포트폴리오
- **글필 사용 패턴:** 여정을 통해 에세이 구조를 배우고, 글쓰기 공간에서 자유롭게 연습함

**페르소나 B — "글쓰기 재개자" 준혁 (31세, 직장인)**

- 브런치에 글을 써보고 싶지만, 작가 승인 절차와 공개 부담이 진입장벽이다.
- 업무 보고서 외에 개인적인 에세이를 써본 적이 없다.
- 원하는 것: 부담 없이 시작하고, 조금씩 나만의 글쓰기 스타일을 찾고 싶다.
- 핵심 동기: 자아 성찰, 정체성 표현, 사이드 프로젝트
- **글필 사용 패턴:** 마음챙김 여정으로 진입 후, 글쓰기 공간에서 글감 미리보기 화면을 통해 글감을 탐색하고 자유 글쓰기 시작

**페르소나 C — "글쓰기 성장자" 지원 (27세, 주니어 마케터)**

- 블로그를 운영 중이지만, 글의 논리적 깊이와 구조에 한계를 느낀다.
- 피드백을 받을 곳이 없어 혼자 정체되어 있다.
- 원하는 것: 내 글의 구조적 약점을 진단받고, 체계적으로 개선하고 싶다.
- 핵심 동기: 커리어 성장, 전문성 구축
- **글필 사용 패턴:** 고급 여정에서 논증 에세이 기법을 습득하고, 글쓰기 공간에서 즉시 적용하며 성장을 체감

### 5.2. 타겟 세그먼트 우선순위

1. **1차 타겟:** 20~30대 글쓰기에 관심 있지만 시작을 못 하는 "잠재 작가"
2. **2차 타겟:** 대학생 (에세이·리포트 역량 향상 필요)
3. **3차 타겟:** 직장인 (업무 커뮤니케이션 및 사이드 프로젝트)

## 6. 제품 비전 및 전략

### 6.1. 제품 비전

> **"여정을 통해 성장하고, 글쓰기로 표현하라."**

글필은 여정이라는 구조화된 학습 경로를 통해 사용자가 글쓰기 역량을 단계적으로 쌓고, 그 성장의 결과물을 자유로운 글쓰기로 발현하는 플랫폼이다. AI는 대필자가 아닌 소크라테스식 코치이며, 사고의 주체는 언제나 사용자 자신이다.

### 6.2. 제품 전략: "Do One Thing Well" (DOTADIW)

글필은 **Opinionated Product** 전략을 따른다. 여러 기능을 나열하는 대신, 하나의 핵심 기능에 모든 역량을 집중하고 나머지는 그것을 지원하는 역할로 명확히 구분한다.

| 기능 계층            | 기능명                | 역할                                        | 사용자 가치                                    |
| -------------------- | --------------------- | ------------------------------------------- | ---------------------------------------------- |
| **Core (핵심)**      | 여정 (Journey)        | 한국어 글쓰기 교육 커리큘럼                 | "단계별로 실력이 쌓이는 구조화된 학습"         |
| **Secondary (부가)** | 글쓰기 (Writing)      | 여정에서 배운 내용을 자유롭게 연습하는 공간 | "배운 것을 부담 없이 적용해보는 나만의 연습장" |
| **Tertiary (보조)**  | 글감 (Writing Prompt) | 글쓰기 내 빈 화면 공포 방지 도구            | "아이디어가 막힐 때 사고를 촉발하는 출발점"    |

**이 위계는 제품의 모든 의사결정 기준이다.** 기능 추가, 화면 설계, 개발 우선순위, 마케팅 메시지 등 모든 판단은 "여정 경험을 더 좋게 만드는가?"를 첫 번째 질문으로 삼는다.

### 6.3. 기능 위계의 UX 원칙

글필의 기능 위계는 정보 구조와 인터랙션 설계에 직접 반영된다.

**여정이 앱의 중심이다:** 홈 화면은 진행 중인 여정을 가장 크고 먼저 보여준다. 하단 내비게이션에서 여정은 독립된 탭으로 존재한다.

**글쓰기는 여정의 출구이자 연습 공간이다:** 글쓰기 탭은 존재하지만, 홈에서의 노출 우선순위는 여정보다 낮다. 여정 세션을 완료하면 글쓰기 공간으로 자연스럽게 연결되는 흐름을 설계한다.

**글감은 글쓰기 진입 시 미리보기 후 에디터 내에서 활용된다 (v2.1 변경):** 글감은 독립 탭이나 섹션으로 존재하지 않는다. 단, 글쓰기 탭을 탭하면 에디터가 바로 열리지 않고 "오늘 어떤 글을 써볼까요?" 중간 화면이 먼저 표시된다. 이 화면에서 글감 카드 몇 장을 미리 볼 수 있으며, 사용자는 글감을 선택하거나 "직접 쓸게요"를 택해 빈 에디터로 진입한다. 이 구조는 Bottom Sheet만으로는 낮은 글감 발견성을 보완하면서, 독립 탭이나 상세 화면을 주는 것보다 위계를 지킨다.

> **v2.1 설계 결정 근거:** v2.0 리스크 평가에서 "Bottom Sheet 하나만으로는 발견성이 너무 낮을 수 있다"는 우려가 제기되었다. 독립 탭 복원(위계 훼손)과 Bottom Sheet 유지(발견성 저하) 사이의 절충점으로 글쓰기 진입 시 1단계 중간 화면을 도입한다. 글감은 여전히 독립 탭이 없으며, 탐색의 주 목적이 아닌 글쓰기의 시작 옵션 중 하나로 위계를 유지한다.

### 6.4. 핵심 가치 제안 (Value Proposition)

| 기존 경험                       | 글필의 제안                                    |
| ------------------------------- | ---------------------------------------------- |
| 글쓰기 앱 = 분량 채우기         | 여정을 통한 역량 성장, 성장이 눈에 보임        |
| 빈 화면 앞에서 막막함           | 여정 → 배움 → 글쓰기 흐름으로 자연스러운 시작  |
| 한 번 쓰고 끝나는 일회성 글쓰기 | 초안 → AI 피드백 → 수정의 다단계 워크플로우    |
| 실력이 느는지 알 수 없음        | 여정 진행률 + 사고력 성장 데이터로 발달 시각화 |
| AI가 대신 써줌 → 실력 퇴화      | AI는 질문자 역할, 사고의 주체는 사용자         |

### 6.5. 설계 원칙

1. **여정 우선 (Journey First):** 모든 화면 설계와 기능 우선순위의 최상위 기준은 "이것이 여정 경험을 강화하는가?"이다.
2. **사고 우선 (Thinking First):** 모든 기능은 "이것이 사용자의 사고를 자극하는가?"를 두 번째 기준으로 삼는다.
3. **명확한 위계 (Clear Hierarchy):** 여정 > 글쓰기 > 글감의 위계는 UI, UX, 마케팅 전반에서 일관되게 유지된다.
4. **점진적 노출 (Progressive Disclosure):** 한 번에 모든 것을 보여주지 않는다. 피드백도, 학습 단계도, 기능도 사용자의 수준에 맞춰 점진적으로 노출한다.
5. **수정은 성장이다 (Revision = Growth):** 초고의 완벽함을 추구하지 않는다. 수정 과정 자체를 긍정적 경험으로 설계한다.
6. **간결함 (Simplicity):** 1인 개발 프로젝트의 제약을 인정하고, 핵심 루프에 집중한다.

### 8.1. 기능 위계 구조도

```
글필 (Geulpil)
│
├── [CORE] 여정 (Journey) — 플랫폼의 핵심
│   ├── 세션 (Session) — 여정 내 학습 노드 (10~20분)
│   │   ├── 스텝: 개념 학습 (Learn)
│   │   ├── 스텝: 예시 읽기 (Read)
│   │   ├── 스텝: 가이드 질문 (Guided Question)
│   │   ├── 스텝: 글쓰기 (Write)
│   │   ├── 스텝: AI 피드백 (Feedback)
│   │   └── 스텝: 수정 (Revise)
│   └── 완료 → 글쓰기 공간으로 자연스럽게 연결
│
├── [SECONDARY] 글쓰기 (Writing) — 여정의 연습 공간
│   ├── 글쓰기 진입 화면 (v2.1 신규) ← "오늘 어떤 글을 써볼까요?" 글감 미리보기
│   ├── 자유 글쓰기 에디터
│   ├── 서재 (Library) — 작성한 글 아카이브
│   └── [TERTIARY] 글감 (Writing Prompt) — 에디터 내 보조 도구
│       ├── 감각형 (Sensory)
│       ├── 회고형 (Reflection)
│       └── 의견형 (Opinion)
│
└── [SYSTEM] 프로필 / 설정
    ├── 성장 통계 (완료 여정, 작성 글 수)
    └── 계정 및 구독 관리
```

## 9. 정보 구조 및 화면 설계

### 9.1. 글로벌 내비게이션 (하단 탭 바)

기능 위계를 그대로 반영한 4탭 구조이다.

```
[ 홈 ]  [ 여정 ]  [ 글쓰기 ]  [ 프로필 ]
```

**탭 설계 근거:**

| 탭     | 대응 기능 계층 | 역할                                                   |
| ------ | -------------- | ------------------------------------------------------ |
| 홈     | Dashboard      | 여정 진행 현황 중심의 대시보드. 사용자를 여정으로 유도 |
| 여정   | Core           | 여정 탐색 및 진행의 전용 공간                          |
| 글쓰기 | Secondary      | 글감 미리보기 진입 화면 + 자유 글쓰기 에디터 + 서재    |
| 프로필 | System         | 성장 통계, 계정, 구독 관리                             |

> **v1과의 차이:** 기존의 홈 화면 내 "글감" 서브탭이 제거되었다. 글감은 더 이상 독립 탐색 영역이 아니며, 글쓰기 탭 진입 시 중간 화면과 에디터 내부의 보조 도구로만 존재한다.

---

### 9.2. 화면 상세 설계

---

#### 9.2.1. 홈 (Home)

홈 화면은 여정 중심의 단일 구조다. 서브탭을 제거하여 인지 부담을 낮추고, 사용자의 시선이 자연스럽게 진행 중인 여정으로 향하도록 설계한다.

**화면 구성:**

- **인사 영역:** "좋은 아침입니다, {사용자명}님." 시간대별 동적 인사
- **[PRIMARY] 현재 진행 중인 여정 카드:** 화면 최상단에 가장 크게 배치. 여정명, 현재 세션명, 진행률(프로그레스 바 + 퍼센트), "이어하기 →" 버튼 포함. 진행 중인 여정이 없으면 "첫 여정 시작하기" CTA를 동일 위치에 표시
- **[PRIMARY] 나의 여정 목록:** 진행 중인 모든 여정 카드 리스트. "여정 더보기 →" 링크로 여정 탭으로 이동
- **[SECONDARY] 오늘의 글쓰기 제안:** 홈 하단에 작은 카드로 배치. "오늘 자유롭게 글을 써보세요" + 글쓰기 탭으로 이동하는 버튼. 글감 제목을 직접 노출하지 않고, 글쓰기 공간으로의 진입만을 유도

> **설계 의도:** 홈에서 글감을 직접 노출하지 않는다. 글감 탐색은 글쓰기 탭 진입 시 중간 화면에서 이루어지며, 홈에서의 노출은 글감을 핵심 기능처럼 보이게 하는 잘못된 위계를 만든다.

---

#### 9.2.2. 여정 (Journey)

여정 탭은 탐색(Discover)과 나의 진행(My Progress)을 하나의 공간에서 처리한다.

**화면 구성:**

- **페이지 타이틀:** "여정"
- **진행 중인 여정 섹션:** 상단 배치. 카드 형태로 현재 참여 중인 여정 목록. 각 카드에 썸네일, 여정명, 현재 세션명, 진행률(프로그레스 바 + 퍼센트) 포함
- **여정 탐색 섹션:** "새 여정 시작하기" 타이틀. 카테고리 필터 칩(전체 / 글쓰기 기술 / 마음챙김 / 실용 글쓰기) + 여정 카드 리스트. 각 카드에 썸네일, 여정명, 설명 텍스트, 세션 수, 잠금/무료 여부 표시

---

#### 9.2.3. 여정 상세 (Journey Detail)

- **헤더:** 뒤로 가기 + 여정 제목 + 더보기(⋮)
- **히어로 이미지:** 여정 대표 이미지
- **여정 제목 및 설명:** 이 여정에서 다루는 내용과 기대 성과
- **전체 진행률 카드:** 완료 세션 수 / 전체 세션 수 + 달성률 % + 세션별 미니 프로그레스 바
- **세션 리스트:** 수직 리스트로 세션을 순서대로 표시. 각 세션 카드에 상태 아이콘(✓ 완료 / 숫자 진행 중 / 🔒 대기 중), 세션 제목, 세션 설명(진행 중인 경우만), "지금 시작하기 →" 버튼(진행 중인 경우만) 포함. 이전 세션 완료 시 다음 세션 잠금 해제되는 선형 진행 구조
- **[연결 포인트] 여정 완료 후 흐름:** 여정의 마지막 세션 완료 시 "글쓰기 공간에서 배운 내용을 자유롭게 표현해보세요" CTA를 통해 글쓰기 탭으로 자연스럽게 연결

---

#### 9.2.4. 세션 진행 (Session Flow)

세션 진행은 전체 화면을 사용하는 스텝 바이 스텝 플로우이다.

**모바일 웹 키보드 대응:**

- 글쓰기(WRITE) 스텝에서 키보드 활성화 시 상단 프로그레스 바는 `position: sticky` 유지
- "제출" 버튼은 키보드 위 고정 (`visualViewport` API 활용)
- CSS 높이 기준: `height: 100dvh` 사용 (dynamic viewport height, iOS 15.4+/Android Chrome 지원)

**화면 구성 (멀티 스텝):**

- **상단 프로그레스 바:** 현재 스텝 / 전체 스텝 수 시각화

| 스텝 유형                         | UI 구성                                                                    |
| --------------------------------- | -------------------------------------------------------------------------- |
| **개념 학습 (Learn)**             | 제목 + 핵심 개념 설명 텍스트 + 예시 카드 + "다음" 버튼                     |
| **예시 읽기 (Read)**              | 모범 에세이 또는 문단 + 분석 포인트 하이라이트 + "다음" 버튼               |
| **가이드 질문 (Guided Question)** | 메타인지 촉발 질문 + 텍스트 입력 필드 + "답변 제출" 버튼                   |
| **글쓰기 (Write)**                | 세션 프롬프트 + 풀스크린 에디터 + 단어 수 카운터 + "제출" 버튼             |
| **AI 피드백 (Feedback)**          | 제출된 글 + AI 피드백 카드(강점/개선점/질문) + "수정하기" 또는 "다음" 버튼 |
| **수정 (Revise)**                 | 피드백이 인라인 표시된 에디터 + "재제출" 버튼                              |

#### 9.2.5. 글쓰기 탭 (Writing Tab)

글쓰기 탭은 두 가지 영역으로 구성된다: 새 글 시작 영역(진입 화면 포함)과 서재(기존 글 관리).

**화면 구성:**

- **페이지 타이틀:** "글쓰기"
- **[PRIMARY CTA] "새 글 쓰기" 버튼:** 화면 상단의 가장 눈에 띄는 위치. 탭하면 글감 미리보기 진입 화면으로 이동 (v2.1 변경)
- **서재 (나의 글):** 작성한 글 카드 리스트. 각 카드에 날짜, 글 제목, 본문 미리보기(2줄), 단어 수, 더보기(⋮) 메뉴
- **검색 바:** "기록된 생각을 검색해보세요"
- **정렬:** 최신순 (기본)

#### 9.2.6. 글쓰기 진입 화면 — 글감 미리보기 (Writing Entry Screen) ✦ v2.1 신규

글쓰기 탭에서 "새 글 쓰기"를 탭하면 에디터가 직접 열리지 않고, 이 중간 화면이 먼저 표시된다.

> **도입 배경:** v2.0 리스크 평가에서 Bottom Sheet만으로는 글감 발견성이 지나치게 낮다는 점이 지적되었다. 독립 탭을 부여하는 대신, 글쓰기 시작 직전의 자연스러운 흐름 내에 글감 탐색 공간을 삽입하는 방식으로 절충한다.

**화면 구성:**

- **화면 타이틀:** "오늘 어떤 글을 써볼까요?"
- **필터 칩:** 전체 / 감각 / 회고 / 의견
- **글감 카드 목록 (3~5장):** 각 카드에 유형 태그(배지), 글감 제목, 글감 본문 요약. **응답 수 (N명이 이 글감으로 글을 썼어요)** 소형 표시 포함 (v2.1 복원)
- **"직접 쓸게요" 버튼:** 글감 없이 빈 에디터로 바로 진입하는 옵션. 화면 하단에 항상 표시

**인터랙션:**

- 글감 카드 탭 → 시트 또는 미리보기 없이 바로 해당 글감이 참조 카드로 고정된 에디터가 열림
- "직접 쓸게요" 탭 → 빈 에디터가 열림

> **설계 의도:** 이 화면은 글감을 탐색하는 공간이지, 글감의 소셜 맥락(다른 사람의 글, 참여 통계 상세)을 보여주는 공간이 아니다. 응답 수는 "823명이 이 글감으로 글을 썼어요"처럼 심리적 안전감을 주는 사회적 증거(Social Proof) 역할에 한정한다. 글감 상세 화면은 존재하지 않는다.

#### 9.2.7. 글 에디터 (Writing Editor)

글쓰기 진입 화면에서 글감을 선택하거나 "직접 쓸게요"를 탭하면 에디터가 열린다.

**화면 구성:**

- **헤더:** 뒤로 가기 + 글 제목(편집 가능) + 더보기(⋮) + 완료(✓) 버튼
- **날짜 표시:** 작성일
- **제목 필드:** 큰 폰트로 편집 가능한 제목
- **[선택적] 글감 참조 카드:** 글감을 선택한 경우 에디터 상단에 고정 표시. 탭하면 접힘/펼침. 글감 없이 진입한 경우 미표시
- **본문 영역:** 충분한 여백의 클린한 텍스트 입력 영역. 산세리프 본문 폰트, 적절한 줄 간격
- **[TERTIARY] "아이디어가 필요하신가요?" 버튼:** 글감 없이 진입한 경우, 본문이 비어 있는 동안만 표시. 탭하면 글감 선택 Bottom Sheet가 올라옴 (글감 미리보기 화면을 거치지 않고 에디터 내에서 추가 탐색 가능한 경로)
- **에디터 기능:** 기본 텍스트 입력 (볼드/이탤릭 정도)

---

#### 9.2.8. 글감 선택 시트 (Writing Prompt Sheet)

에디터 내에서 "아이디어가 필요하신가요?" 버튼을 탭했을 때 올라오는 Bottom Sheet이다. 글쓰기 진입 화면에서 글감을 선택하지 않고 에디터에 진입한 사용자를 위한 보조 경로다.

**화면 구성:**

- **시트 타이틀:** "어떤 이야기를 꺼내볼까요?"
- **필터 칩:** 전체 / 감각 / 회고 / 의견
- **글감 카드 리스트:** 각 카드에 유형 태그, 글감 제목, 글감 본문 요약, 응답 수 소형 표시. 탭하면 시트가 닫히고 에디터 상단에 해당 글감이 참조 카드로 고정됨
- **"직접 쓸게요" 버튼:** 글감 없이 빈 에디터로 돌아가는 옵션

---

#### 9.2.9. 글 상세 / 읽기 뷰 (Writing Detail)

- **헤더:** 뒤로 가기 + 글 제목 + 더보기(⋮)
- **날짜:** 작성일
- **제목:** 큰 타이포그래피
- **저자명:** 우측 정렬
- **Writing Prompt 참조 카드 (선택적):** 이 글이 글감을 사용해 작성된 경우에만 표시. 접힘/펼침(Accordion) UI로 글감 제목과 본문 표시. 글감 없이 작성된 글에는 이 카드가 노출되지 않음
- **본문:** 에세이 본문 전체

---

#### 9.2.10. 프로필 / 설정 (Profile / Settings)

- **페이지 타이틀:** "설정"
- **프로필 영역:** 아바타, 사용자명, 이메일
- **성장 통계 카드:** 완료한 여정 수 / 작성한 글 수 (여정이 우선 표시)
- **계정 관리:** 계정 보안, 알림 설정
- **접근성:** 글꼴 크기 (보통/크게/매우 크게), 동작 줄이기 모드
- **서비스 및 데이터:** 구독 관리 (Pro), 데이터 내보내기
- **로그아웃**

## 10. 기능 요구사항 (EARS 표기법)

> EARS (Easy Approach to Requirements Syntax) 표기법을 사용한다.
> 패턴: Ubiquitous / Event-Driven / Unwanted Behavior / State-Driven / Optional

### 10.1. 온보딩 및 계정

| ID          | 유형         | 요구사항                                                                                                                                           |
| ----------- | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| REQ-ACC-001 | Event-Driven | **WHEN** 사용자가 소셜 로그인(Google/Kakao) 버튼을 탭하면, **THE SYSTEM SHALL** OAuth 인증을 수행하고 계정을 생성 또는 연결한다.                   |
| REQ-ACC-002 | Event-Driven | **WHEN** 사용자가 온보딩에서 관심 분야와 글쓰기 경험을 선택하면, **THE SYSTEM SHALL** 해당 수준에 맞는 여정을 첫 번째 추천 여정으로 홈에 표시한다. |
| REQ-ACC-003 | Event-Driven | **WHEN** 사용자가 "데이터 내보내기"를 요청하면, **THE SYSTEM SHALL** 사용자의 모든 글을 마크다운 파일로 패키징하여 다운로드 링크를 제공한다.       |

| ID           | 유형         | 요구사항                                                                                                                                                               |
| ------------ | ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| REQ-HOME-002 | State-Driven | **WHILE** 사용자가 진행 중인 여정이 없는 상태에서, **THE SYSTEM SHALL** 홈 화면 최상단에 "첫 여정 시작하기" CTA를 표시한다.                                            |
| REQ-HOME-003 | Ubiquitous   | **THE SYSTEM SHALL** 홈 화면 하단에 글쓰기 탭으로 이동하는 "오늘 자유롭게 글을 써보세요" 보조 카드를 표시한다. 이 카드에 글감 제목을 직접 노출하지 않는다.             |

### 10.3. 여정 (Journey) — Core 기능

| ID          | 유형         | 요구사항                                                                                                                                                                            |
| ----------- | ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| REQ-JRN-001 | Ubiquitous   | **THE SYSTEM SHALL** 여정을 글쓰기 기술/마음챙김/실용 글쓰기의 3가지 카테고리로 분류하고, 각 카테고리에 해당하는 필터 칩을 여정 탭에서 제공한다.                                    |
| REQ-JRN-002 | Event-Driven | **WHEN** 사용자가 여정 카드를 탭하면, **THE SYSTEM SHALL** 여정 상세 화면(설명, 진행률, 세션 리스트)을 표시한다.                                                                    |
| REQ-JRN-003 | Ubiquitous   | **THE SYSTEM SHALL** 여정 내 세션들을 선형 진행 구조로 배치하며, 이전 세션이 완료되어야 다음 세션이 잠금 해제된다.                                                                  |
| REQ-JRN-004 | State-Driven | **WHILE** 세션이 "대기 중" 상태인 동안, **THE SYSTEM SHALL** 해당 세션 카드에 잠금(🔒) 아이콘을 표시하고 탭 상호작용을 비활성화한다.                                                |
| REQ-JRN-005 | Event-Driven | **WHEN** 사용자가 "지금 시작하기" 버튼을 탭하면, **THE SYSTEM SHALL** 해당 세션의 첫 번째 스텝을 전체 화면 플로우로 시작한다.                                                       |
| REQ-JRN-006 | Event-Driven | **WHEN** 사용자가 여정의 마지막 세션을 완료하면, **THE SYSTEM SHALL** 여정 완료 축하 화면을 표시하고, "글쓰기 공간에서 배운 내용을 표현해보세요" CTA를 통해 글쓰기 탭으로 연결한다. |

### 10.4. 세션 및 스텝 (Session & Step)

| ID          | 유형         | 요구사항                                                                                                                                                      |
| ----------- | ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| REQ-SES-001 | Ubiquitous   | **THE SYSTEM SHALL** 세션 진행 시 상단에 프로그레스 바(현재 스텝/전체 스텝)를 표시한다.                                                                       |
| REQ-SES-002 | Event-Driven | **WHEN** 사용자가 "개념 학습" 스텝에서 "다음" 버튼을 탭하면, **THE SYSTEM SHALL** 다음 스텝으로 전환한다.                                                     |
| REQ-SES-003 | Event-Driven | **WHEN** 사용자가 "가이드 질문" 스텝에서 답변을 입력하고 제출하면, **THE SYSTEM SHALL** 답변을 저장하고 다음 스텝으로 진행한다.                               |
| REQ-SES-004 | Event-Driven | **WHEN** 사용자가 "글쓰기" 스텝에서 글을 제출하면, **THE SYSTEM SHALL** AI 분석을 수행하고 "AI 피드백" 스텝으로 전환한다.                                     |
| REQ-SES-005 | Event-Driven | **WHEN** AI 피드백 스텝에서 사용자가 "수정하기" 버튼을 탭하면, **THE SYSTEM SHALL** 피드백 내용이 참조 가능한 수정 에디터를 표시한다.                         |
| REQ-SES-006 | Event-Driven | **WHEN** 사용자가 수정된 글을 재제출하면, **THE SYSTEM SHALL** 수정 전후를 비교 분석하고 개선된 부분을 하이라이트하여 피드백한다.                             |
| REQ-SES-007 | Unwanted     | **IF** 세션 진행 중 앱이 예기치 않게 종료되면, **THE SYSTEM SHALL** 마지막으로 완료된 스텝까지의 진행 상태를 자동 저장하고, 재진입 시 해당 스텝부터 재개한다. |
| REQ-SES-008 | State-Driven | **WHILE** 사용자가 글쓰기 스텝에서 글을 작성하는 동안, **THE SYSTEM SHALL** 5초 간격으로 자동 저장을 수행한다.                                                |

### 10.5. 글쓰기 (Writing) — Secondary 기능

| ID          | 유형         | 요구사항                                                                                                                                                                                                                           |
| ----------- | ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| REQ-WRT-001 | Event-Driven | **WHEN** 사용자가 글쓰기 탭에서 "새 글 쓰기" 버튼을 탭하면, **THE SYSTEM SHALL** 글쓰기 진입 화면("오늘 어떤 글을 써볼까요?")을 표시하고 글감 카드 목록을 로드한다. _(v2.1 변경: 에디터 직접 진입 → 글감 미리보기 중간 화면 진입)_ |

| REQ-WRT-003 | Event-Driven | **WHEN** 사용자가 글쓰기 진입 화면에서 "직접 쓸게요"를 탭하면, **THE SYSTEM SHALL** 글감 없이 빈 에디터를 열고, 본문 영역 하단에 "아이디어가 필요하신가요?" 버튼을 표시한다.                                                       |
| REQ-WRT-004 | State-Driven | **WHILE** 에디터 본문이 비어 있고 글감이 선택되지 않은 동안, **THE SYSTEM SHALL** "아이디어가 필요하신가요?" 버튼을 표시한다. 본문에 내용이 입력되면 이 버튼을 숨긴다.                                                             |
| REQ-WRT-005 | Event-Driven | **WHEN** 사용자가 에디터 내 "아이디어가 필요하신가요?" 버튼을 탭하면, **THE SYSTEM SHALL** 글감 선택 Bottom Sheet를 표시한다.                                                                                                      |
| REQ-WRT-006 | State-Driven | **WHILE** 사용자가 글 에디터에서 글을 작성하는 동안, **THE SYSTEM SHALL** 5초 간격으로 자동 저장을 수행하고 저장 상태를 헤더에 표시한다.                                                                                           |
| REQ-WRT-007 | Event-Driven | **WHEN** 사용자가 완료(✓) 버튼을 탭하면, **THE SYSTEM SHALL** 글을 서재에 저장하고 글 상세(읽기 뷰) 화면으로 전환한다.                                                                                                             |
| REQ-WRT-008 | Event-Driven | **WHEN** 사용자가 서재에서 검색 바에 키워드를 입력하면, **THE SYSTEM SHALL** 제목 또는 본문에 해당 키워드가 포함된 글만 필터링하여 표시한다.                                                                                       |

### 10.6. 글감 (Writing Prompt) — Tertiary 기능

| ID          | 유형         | 요구사항                                                                                                                                                                                   |
| ----------- | ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| REQ-PRM-001 | Ubiquitous   | **THE SYSTEM SHALL** 글감을 글쓰기 진입 화면과 에디터 내 Bottom Sheet에서만 표시한다. 독립 탭, 홈 화면, 여정 화면에는 글감을 노출하지 않는다.                                              |
| REQ-PRM-002 | Ubiquitous   | **THE SYSTEM SHALL** 글감 카드에 유형 태그(감각/회고/의견), 글감 제목, 글감 본문 요약, 응답 수(N명이 이 글감으로 글을 썼어요)를 소형 표시한다. _(v2.1 변경: responseCount 소형 표시 복원)_ |
| REQ-PRM-003 | Ubiquitous   | **THE SYSTEM SHALL** 글감에 독립적인 상세 화면을 제공하지 않는다. 글감은 카드 형태로만 존재한다.                                                                                           |
| REQ-PRM-004 | Event-Driven | **WHEN** 사용자가 글감 필터 칩(전체/감각/회고/의견)을 탭하면, **THE SYSTEM SHALL** 해당 유형의 글감만 목록에 표시한다.                                                                     |

### 10.7. AI 피드백 (AI Feedback)

| ID         | 유형         | 요구사항                                                                                                                                                                         |
| ---------- | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| REQ-AI-001 | Event-Driven | **WHEN** 사용자가 글쓰기 스텝에서 글을 제출하면, **THE SYSTEM SHALL** 10초 이내에 소크라테스식 AI 피드백 카드(강점 1~2개, 개선점 1~2개, 사고 촉발 질문 1개)를 생성하여 표시한다. |
| REQ-AI-002 | Ubiquitous   | **THE SYSTEM SHALL** AI 피드백에서 완성된 문장이나 대체 표현을 직접 제시하지 않는다. 방향 힌트와 질문만을 제공한다.                                                              |
| REQ-AI-003 | Ubiquitous   | **THE SYSTEM SHALL** 사용자의 글쓰기 수준(온보딩 선택 + 진행 이력)에 따라 피드백 깊이와 어조를 조절한다.                                                                         |

| REQ-AI-005 | Event-Driven | **WHEN** 사용자가 수정본을 재제출하면, **THE SYSTEM SHALL** 초안 대비 개선된 부분을 명시적으로 언급하여 성장을 강화한다.                                                         |
| REQ-AI-006 | Unwanted     | **IF** AI 분석이 실패하거나 타임아웃되면, **THE SYSTEM SHALL** "분석에 시간이 조금 더 필요합니다" 메시지를 표시하고 재시도 버튼을 제공한다.                                      |

```gherkin
    When 사용자가 글꼴 크기를 "크게"로 변경한다
    Then 앱 전체의 본문 텍스트가 즉시 더 큰 크기로 렌더링된다

  Scenario: 구독 상태 확인
    Given 사용자가 Pro 구독 중이다
    When 사용자가 "구독 관리 (Pro)" 항목을 탭한다
    Then 현재 구독 상태 "ACTIVE"와 갱신일, 구독 변경 옵션이 표시된다

  Scenario: 데이터 내보내기
    Given 사용자가 설정 화면에 있다
    When 사용자가 "데이터 내보내기"를 탭한다
    Then 확인 다이얼로그가 표시된다
    And 사용자가 확인하면 모든 글이 마크다운 파일로 변환되어 다운로드된다
```

## 12. AI 통합 전략

### 12.1. 핵심 원칙: 소크라테스식 코칭

AI는 **완성된 텍스트를 생성하지 않는다.** 대신 다음의 역할을 수행한다.

| 역할       | 설명                           | 예시                                                                             |
| ---------- | ------------------------------ | -------------------------------------------------------------------------------- |
| **진단자** | 글의 구조적/논리적 특성을 분석 | "이 글은 3개의 문단으로 구성되어 있고, 중심 논지가 두 번째 문단에서 전환됩니다." |
| **질문자** | 사고를 확장하는 질문 제시      | "이 주장에 동의하지 않는 사람은 어떤 반론을 펼칠까요?"                           |
| **격려자** | 개선된 부분을 구체적으로 칭찬  | "초안보다 근거가 구체적이 되었고, 특히 두 번째 예시가 설득력을 높였습니다."      |
| **안내자** | 다음 단계를 위한 힌트 제공     | "결론에서 서론의 질문을 다시 한번 떠올려 보는 건 어떨까요?"                      |

### 12.2. 피드백 구조

```
AI 피드백 카드
├── 강점 (1~2개)
│   └── 구체적으로 잘된 부분과 이유
├── 개선점 (1~2개)
│   └── 문제 지적 + 방향 힌트 (답을 주지 않음)
└── 사고 촉발 질문 (1개)
    └── 글의 깊이를 확장하는 메타인지 질문
```

### 12.3. 수준별 피드백 전략

| 수준 | 강점 : 개선점 비율 | 피드백 깊이      | 메타인지 질문 유형                                           |
| ---- | ------------------ | ---------------- | ------------------------------------------------------------ |
| 입문 | 2 : 1              | 문장/문단 단위   | "이 문장에서 가장 하고 싶은 말은?"                           |
| 중급 | 1 : 1              | 논리 구조 단위   | "이 근거가 주장을 충분히 뒷받침하나요?"                      |
| 상급 | 1 : 2              | 에세이 전체 단위 | "이 글의 전제에 동의하지 않는 독자를 어떻게 설득하시겠어요?" |

### 12.4. AI가 하지 않는 것

- 완성된 문장이나 문단을 생성하지 않는다.
- 대체 표현을 직접 제시하지 않는다.
- 맞춤법/문법 교정을 자동으로 적용하지 않는다 (지적은 하되, 수정은 사용자 몫).
- 글의 주제나 방향을 결정하지 않는다.
- 글감을 자동으로 선택하거나 강요하지 않는다.

## 13. 게이미피케이션 설계

### 13.1. 리텐션 메커니즘

게이미피케이션의 중심은 여정 진행 경험이다. 모든 보상 구조는 여정 완료를 최우선 목표로 설계된다.

| 메커니즘                        | 설명                                                              | 구현 우선순위 |
| ------------------------------- | ----------------------------------------------------------------- | ------------- |
| **여정 진행률**                 | 세션별/여정별 프로그레스 바와 달성률                              | P0 (MVP)      |
| **세션 달성 보상**              | 세션 완료 시 시각적 축하 애니메이션                               | P0 (MVP)      |
| **글감 응답 수 (Social Proof)** | 글감 카드에 응답 수 소형 표시 — "823명이 이 글감으로 글을 썼어요" | P0 (MVP)      |
| **글쓰기 스트릭**               | 연속 글쓰기 일수 추적 (여정 세션 또는 자유 글쓰기 모두 인정)      | P1            |
| **단어 수 누적**                | 총 작성 단어 수 카운터                                            | P1            |
| **여정 완료 뱃지**              | 여정 완료 시 프로필에 뱃지 추가                                   | P2            |

### 13.2. 동기 부여 전략

**내재적 동기 (Intrinsic):** AI 피드백을 통한 실력 향상 체감, 여정 진행에 따른 역량 성장 시각화, 자신의 글 아카이브 축적, 메타인지 질문을 통한 지적 자극.

**외재적 동기 (Extrinsic):** 여정 진행률과 달성률, 글감의 응답 수를 통한 사회적 증거(심리적 안전감), 스트릭 시각화, 여정 완료 뱃지를 통한 성취감.

> **v2.1 설계 결정:** 글감의 응답 수(responseCount)는 소셜 기능이 아닌 "사회적 증거(Social Proof)" 역할로 제한한다. "이 글감으로 823명이 글을 썼어요"라는 표시는 글쓰기 입문자에게 심리적 안전망을 제공한다. 이 숫자는 글감 카드 내 소형 표시에 국한하며, 상세 화면, 참여자 목록, 소셜 비교 등의 기능과 연결하지 않는다.

### 14.2. 핵심 데이터 모델

기능 위계를 반영하여 `WritingPrompt`는 `Writing`에 종속된 선택적 연결로 설계한다.

```
User
├── id, email, name, avatarUrl
├── onboardingData (interests, level)
├── subscriptionTier (FREE | PRO)
└── createdAt, updatedAt

-- [CORE] 여정 관련 모델 --

Journey
├── id, title, description, category
├── thumbnailUrl, sessionCount
└── sessions[] → Session

Session
├── id, journeyId, order, title, description
├── steps[] → Step
└── estimatedMinutes

Step
├── id, sessionId, order
├── type (LEARN | READ | GUIDED_QUESTION | WRITE | FEEDBACK | REVISE)
├── content (JSON - 유형별 구조화된 콘텐츠)
└── metadata

UserJourneyProgress
├── userId, journeyId
├── currentSessionOrder, completionRate
└── status (IN_PROGRESS | COMPLETED)

UserSessionProgress
├── userId, sessionId
├── currentStepOrder
├── status (LOCKED | IN_PROGRESS | COMPLETED)
└── stepResponses (JSON - 각 스텝별 사용자 응답)

-- [SECONDARY] 글쓰기 관련 모델 --

Writing
├── id, userId
├── promptId (optional) ← WritingPrompt와의 연결은 선택적
├── sessionId (optional) ← 여정 세션과의 연결은 선택적
├── title, body, wordCount
├── versions[] → WritingVersion
└── createdAt, updatedAt

WritingVersion
├── id, writingId, versionNumber
├── body, wordCount
├── aiFeedback (JSON)
└── createdAt

-- [TERTIARY] 글감 모델 --

WritingPrompt
├── id, title, body
├── type (SENSORY | REFLECTION | OPINION)
├── responseCount  ← v2.1 복원: 카드 내 소형 표시용 (사회적 증거 역할)
└── createdAt
```

> **모델 설계 의도:**
>
> `responseCount` 필드가 v2.1에서 복원되었다. v2.0에서 완전 제거되었으나, 리스크 평가 결과 이 숫자가 글쓰기 입문자에게 제공하는 "나만 쓰는 게 아니다"라는 심리적 안전망 효과가 크다고 판단했다. 단, 이 필드는 글감 카드의 소형 텍스트 표시("N명이 이 글감으로 글을 썼어요")에만 사용하며, 소셜 참여 기능(참여자 목록, 타인 글 열람, 비교) 구현에는 사용하지 않는다.
>
> `partnerId` 필드는 v2.0에서 제거한 결정을 유지한다. 1인 개발 환경에서 브랜드 파트너십 운영은 개발 복잡도 대비 실현 가능성이 낮다.

## 17. 로드맵

### Phase 1 — MVP (0~3개월)

**목표:** 여정 핵심 루프 검증

- 온보딩 (관심사 + 수준 파악)

- 세션 진행 플로우 (전체 스텝 유형): Learn / Read / Guided Question / Write / Feedback / Revise
- AI 피드백 (소크라테스식, 수준별)
- 글쓰기 탭: 글쓰기 진입 화면(글감 미리보기) + 자유 글쓰기 에디터 + 서재
- 글감 카드 (진입 화면 + Bottom Sheet) + responseCount 소형 표시
- 홈: 여정 진행 현황 대시보드
- 여정 진행률 + 세션 완료 애니메이션 (P0 게이미피케이션)
- 인증 (Google/Kakao 소셜 로그인)

### Phase 2 — Growth (3~6개월)

**목표:** 리텐션 강화 + 콘텐츠 확장

- 여정 콘텐츠 확대 (카테고리별 3개 이상)
- 글쓰기 스트릭 + 단어 수 누적 게이미피케이션
- 오프라인 지원 + 자동 동기화
- 개인화 강화 (AI 기반 여정 추천)
- Pro 구독 결제 (인앱 결제)

### Phase 3 — Scale (6~12개월)

**목표:** Pro 수익화 + 사용자 성장 시각화

- 여정 완료 뱃지 + 프로필 포트폴리오
- 사고력 성장 대시보드 (글 길이 추이, AI 피드백 점수 변화)
- 여정 콘텐츠 지속 추가
- 접근성 강화 (WCAG 2.1 AA)

## 부록 A: 용어 정의

| 용어                       | 계층      | 정의                                                                                    |
| -------------------------- | --------- | --------------------------------------------------------------------------------------- |
| 여정 (Journey)             | Core      | 특정 글쓰기 역량 향상을 목표로 한 구조화된 커리큘럼. 글필의 핵심 기능                   |
| 세션 (Session)             | Core      | 여정 내 하나의 학습 노드. 10~20분 완결형                                                |
| 스텝 (Step)                | Core      | 세션 내 하나의 마이크로 활동 단위                                                       |
| 글쓰기 (Writing)           | Secondary | 여정에서 배운 내용을 자유롭게 연습하는 공간. 서재 포함                                  |
| 서재 (Library)             | Secondary | 사용자가 작성한 모든 글의 아카이브. 글쓰기 탭 내 포함                                   |
| 글쓰기 진입 화면           | Secondary | 글쓰기 탭에서 "새 글 쓰기" 탭 시 표시되는 중간 화면. 글감 미리보기 + "직접 쓸게요" 옵션 |
| 글감 (Writing Prompt)      | Tertiary  | 글쓰기 진입 화면과 에디터 내 Bottom Sheet에서만 접근 가능한 아이디어 제공 카드          |
| 응답 수 (responseCount)    | Tertiary  | 특정 글감으로 글을 쓴 사용자 수. 사회적 증거 역할로 카드 내 소형 표시                   |
| 소크라테스식 코칭          | System    | AI가 답을 주지 않고 질문을 통해 사용자 스스로 사고하도록 유도하는 방식                  |
| 메타인지                   | System    | 자신의 사고 과정을 인식하고 조절하는 능력                                               |
| EARS                       | System    | Easy Approach to Requirements Syntax. 요구사항 표기 패턴                                |
| DOTADIW                    | Strategy  | Do One Thing And Do It Well. 글필의 제품 전략 원칙                                      |
| 사회적 증거 (Social Proof) | UX        | "많은 사람이 이미 하고 있다"는 정보를 통해 사용자의 행동 결정을 돕는 심리적 메커니즘    |

| 기능 요구사항 REQ-PRM-002     | responseCount 미노출                                   | 글감 카드에 responseCount 소형 표시                             | 복원 결정 반영                                                                  |
| KPI: 글감 활용률              | 단일 지표 (Bottom Sheet 활용률)                        | 진입 화면 선택률 + Bottom Sheet 활용률 분리 측정                | v2.1 변경 효과를 정량 검증하기 위한 분리 측정                                   |
