# Changelog Policy

## 무엇을 기록하는가

- 제품 원칙 변경
- 핵심 사용자 흐름 변경
- 앱 또는 패키지 책임 변경
- 운영, 보안, 릴리스 정책 변경

## 어디에 기록하는가

- 구조 변경은 해당 섹션의 기준 문서를 먼저 갱신합니다.
- 큰 방향 전환은 상위 `README.md`와 관련 문서를 함께 갱신합니다.
- 폐기된 문서는 `99-archive`로 옮기고 대체 문서를 명시합니다.

## 기록 방식

- 문서는 회의록처럼 시간순으로 누적하지 않습니다.
- 기준 문서는 항상 최신 상태를 유지합니다.
- 시간순 기록이 필요하면 이슈, PR, 커밋에서 추적하고 문서에는 현재 상태만 남깁니다.
