목표: 에디터 내에 AI 기능 구현.

## 사용자의 AI 기능 사용 방식

이 플랫폼의 철학("사유를 끌어올리는 AI")에 가장 잘 맞는 조합은 **방식 2 + 방식 6의 레이어드 구조**입니다.

AI 버튼: 하단 고정 툴바에 AI 버튼을 추가합니다. 문장을 드래그한 상태에서 해당 버튼을 누르면 레이어 1이 되며, 선택하지 않고 누르면 레이어2가 됩니다.

**Layer 1 — 세밀한 편집 (방식 2)**
문장/단어 드래그 → 하단 AI 버튼 강조 → 탭 → 바텀 시트에서 어휘다듬기/맞춤법/표현개선/톤조정어휘다듬기/맞춤법/표현개선/톤조정 선택 → 원문 유지 OR 적용 선택

**Layer 2 — 전체 검토 (방식 6)**
AI 버튼 탭 (텍스트 비선택 상태) → "전체 검토" 버튼 → 문서 전체 맞춤법/띄어쓰기 인라인 하이라이트 표시 → 개별 수락/거절

이 구조는 **"AI가 글을 대신 쓰지 않고, 사용자가 쓴 글을 다듬는 도구"**라는 원칙을 모든 레이어에서 유지하며, 흐름을 최대한 보호하면서도 필요할 때만 AI가 등장하는 UX를 만들 수 있습니다.

### 방식 2 — 텍스트 선택 → 하단 툴바 AI 버튼 활성화 → 탭 → 바텀 시트

텍스트를 드래그하면 하단 툴바의 AI 버튼이 강조(색상 변화, 진동)되어 "선택 중" 상태임을 알린다. 그 버튼을 탭하면 바텀 시트가 올라오며 기능 목록이 보인다.

> 예: "드래그 → 하단 AI 버튼 빛남 → 탭 → 바텀 시트에서 기능 선택"

**장점:** 모바일에 가장 최적화된 공간 사용, 오탭 없음, 선택 상태 시각 피드백 제공
**단점:** 2단계 터치 필요 (드래그 후 손가락을 하단까지 이동)

### 방식 6 — 전체 문서 AI 리뷰 모드

AI 버튼 → "전체 검토" 선택 시 문서 전체를 분석하여 수정 제안 부분을 인라인 하이라이트로 표시한다. 각 하이라이트를 탭하면 제안과 수락/거절 버튼이 나온다.

**장점:** 맞춤법/띄어쓰기 일괄 검사에 탁월, 전체 흐름 검토 후 쓰기에 적합
**단점:** 집필 중이 아닌 "검토 단계"에서만 유효, 실시간 편집 흐름과 분리됨

## 필요한 기능

### 판단 기준

기능을 추가하기 전, 모든 기능에 아래 필터를 통과시켰습니다.

> **"이 기능은 사용자 대신 생각하는가, 아니면 사용자가 더 잘 생각하게 돕는가?"**

첫 번째라면 제외, 두 번째라면 검토 대상입니다. Authorship AI의 설계 원칙과 동일한 접근이며, 이들은 AI가 답을 주는 능력 자체를 제거하는 방식으로 철학적 일관성을 유지했습니다.

기능은 성격에 따라 두 유형으로 나뉩니다. **"교정형"**은 정답이 존재하는 교정 작업이고, **"코칭형"**은 정답 없이 사용자의 사유를 더 깊게 이끄는 작업입니다.

### 교정형 — Layer 1 (단어 ~ 문장 수준)

가장 명백하게 포함되어야 할 기능. 창의성이나 사유와 무관한 순수 규칙 교정이며 흐름을 방해하지 않는다. 다만 결과를 자동 적용하지 않고 반드시 사용자 확인 후 수락하는 구조로 구현해야 한다.

문장 드래그 -> AI 버튼 클릭 -> 바텀시트가 올라옴 -> 원하는 기능 클릭 -> AI는 검토 후 최대 3개의 제안을 생성하여 바텀시트 내에 표시한다. 수락을 클릭하면 적용이 되어 내용이 변경된다.

**1. 어휘 & 표현 개선**
선택한 단어/구문에 대해 의미상 더 정확하거나 자연스러운 한국어 대안 1~3개를 제안한다. 핵심 제약: AI가 "더 좋은 단어"를 단정짓지 않고 사용자가 고르는 구조. 사용자의 **목소리(voice)를 보존**하는 것이 최우선이며, 제안만 한다.​

**2. 문장 명료화**
"이 문장이 두 가지로 해석될 수 있습니다"처럼 모호성을 *지적*하고 더 명확한 표현을 제안한다. AI가 재작성하는 것이 아니라, 원문의 의도를 유지하면서 명확도를 높이는 방향으로만 작동해야 한다.

**3. 문장 호흡 조정**
에세이에서 리듬은 중요하다. 너무 긴 문장(숨이 차는 구조)이나 너무 짧은 문장이 반복될 때 분리/합치기를 제안한다. 단, 스타일로서 의도적으로 짧은 문장을 반복하는 경우가 있으므로 **자동 수정 금지**, 제안만 표시.

---

### Layer 2 (전체 문서 수준 검토)

**5. 전체 문서 맞춤법 & 중복 표현 일괄 검토**
작성 완료 후 전체 문서에서 맞춤법, 띄어쓰기, 그리고 같은 단어/표현이 불필요하게 반복되는 부분을 인라인 하이라이트로 표시한다. 각 항목을 개별 수락/거절할 수 있어야 한다. 이 기능은 "출판"이 아닌 "다듬기"를 위한 것이므로 철학에 부합한다.​

**6. 문단 연결 흐름 피드백**
전체 검토 시 문단 간 논리 연결이 끊기는 지점을 감지해 "여기서 A 문단과 B 문단의 연결이 갑작스럽습니다"처럼 *지점만 표시*한다. 어떻게 연결할지는 사용자가 결정한다. 이는 구조를 대신 고쳐주는 것이 아닌, 독자 시점에서의 피드백이다.
