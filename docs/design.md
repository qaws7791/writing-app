이 플랫폼은 **교육 서비스의 얼굴을 하고 있지만, UX는 “수업 앱”이 아니라 “매일 들어오고 싶어지는 개인 글쓰기 앱”**으로 설계해야 합니다.

방향은 명확합니다.
**홈은 듀오링고처럼 오늘 해야 할 한 가지를 제시하고, 글감 찾기는 Threads·당근처럼 피드형 탐색을 주고, 학습 활동은 Quill/NoRedInk처럼 작은 인터랙션 단위로 쪼개며, 글쓰기는 토스처럼 복잡한 기능을 카드·바텀시트·단계형 흐름으로 단순화**하는 구조가 가장 적합합니다. Threads는 관심사 중심의 주제 태그·커스텀 피드·커뮤니티를 강화하고 있고, Duolingo는 bite-sized lesson과 streak·daily quest 같은 반복 동기를 중심에 두며, Toss는 한 화면에서 핵심 상태를 빠르게 파악하게 하는 “at a glance” 홈 구조를 강하게 밀고 있습니다. ([페이스북][1])

## 1. 전체 제품 구조

바텀 네비게이션은 말씀하신 대로 **5개로 고정**합니다.

- 홈
- 글감 찾기
- 학습 활동
- 글쓰기
- 프로필

이 5개는 역할이 절대 겹치면 안 됩니다.

- **홈**은 “오늘 무엇을 할지 결정하는 곳”
- **글감 찾기**는 “무엇에 대해 쓸지 발견하는 곳”
- **학습 활동**은 “짧은 훈련을 하는 곳”
- **글쓰기**는 “실제로 쓰는 곳”
- **프로필**은 “내 기록을 돌아보는 곳”

즉, 탐색, 훈련, 작성, 회고를 분리해야 합니다.

## 2. 전체 UI/UX 원칙

### 2-1. 최신 앱 스타일을 반영하는 기본 톤

Threads, 당근, 토스, 듀오링고에서 공통적으로 배울 수 있는 건 다음입니다.

- 첫 화면에서 해야 할 일이 복잡하지 않다
- 카드와 피드 중심으로 콘텐츠를 가볍게 소비한다
- 필터는 칩(chip)과 바텀시트로 짧게 연다
- 한 번에 한 가지 핵심 액션만 강하게 밀어준다
- 사용자의 관심사, 상태, 최근 행동에 따라 개인화한다
- 길고 무거운 폼보다 짧고 연속적인 상호작용을 선호한다 ([페이스북][1])

그래서 이 플랫폼의 UX 원칙은 이렇게 잡는 것이 좋습니다.

**첫째, 홈에서는 결정 피로를 줄인다.**
사용자가 앱을 켰을 때 “무엇을 해야 하지?”를 생각하게 하면 안 됩니다. 오늘의 추천 학습, 이어서 하기, 추천 글감, 최근 초안을 바로 보여줘야 합니다.

**둘째, 탐색은 피드형으로 만든다.**
글감은 카탈로그보다 피드가 낫습니다. Threads의 관심 주제, custom feed처럼 주제 기반 탐색 경험이 강할수록 글쓰기 진입 장벽이 내려갑니다. ([페이스북][1])

**셋째, 학습은 짧고 반복적으로 만든다.**
Duolingo와 NoRedInk가 공통적으로 보여주는 것은, 복잡한 개념도 step-by-step과 작은 체크포인트로 나눌 때 유지율이 높아진다는 점입니다. ([Duolingo Blog][2])

**넷째, 복잡한 기능은 한 화면에 다 노출하지 않는다.**
토스처럼 메인 화면은 요약 카드 중심으로 두고, 상세 설정·고급 옵션은 바텀시트나 단계형 패널로 분리하는 편이 좋습니다. Material 3도 모바일에서 긴 옵션 목록은 메뉴보다 bottom sheet로 적응시키는 방식을 권장합니다. ([App Store][3])

### A. 홈

홈은 “대시보드”가 아니라 **오늘의 행동 유도 화면**이어야 합니다.

- 오늘 무엇을 할지 결정하게 하기
- 중단했던 흐름을 바로 이어가게 하기
- 매일 들어오게 만드는 루틴 제공

#### 화면 구조

1. 상단 인사 영역
   - 인삿말
2. 오늘의 영감 4개
3. 내가 쓴 글, 저장한 글감(탭 구조)
   1. 내가 쓴글 목록: 제목, 내용, 마지막 수정일자, 글자수
   2. 저장한 글감 목록: 글감 제목, 글감 주제, 글감 난이도

### B. 글감 찾기

이 탭은 Threads와 당근에서 배워야 합니다.
Threads는 topics, custom feeds, communities를 통해 관심사 중심 탐색을 강화했고, Karrot은 카테고리와 지역/맥락 기반으로 빠르게 둘러보는 경험이 강합니다. 이 탭은 검색형 라이브러리보다 **피드형 탐색 + 관심사 기반 추천**이 핵심입니다. ([페이스북][1])

#### 글감 찾기의 핵심 목적

- 막막함 해소
- 흥미 유발
- 바로 쓰기로 연결

#### 정보 구조

상단:

- 검색창
- 필터 버튼
- 관심 주제 칩

본문:

- For You
- 인기 글감
- 짧게 쓰기 좋은 글감
- 깊게 쓰기 좋은 글감
- 최근 관심 주제 기반 추천
- 저장한 글감

#### 카드 타입

**1. 단문 글감 카드**

- 한 문장 프롬프트
- 예상 글 길이: 짧음/보통/깊이
- 추천 레벨
- 감정 태그 또는 사고 태그

예:

- “최근에 내 생각이 바뀐 순간은?”
- “사람들은 왜 익숙한 것을 떠나기 어려울까?”

**2. 구조형 글감 카드**

- 글의 목적 포함
- 추천 아웃라인 포함
- argumentative / reflective / explanatory 표시

**3. 시리즈형 글감**

- 하나의 대주제 아래 여러 질문
  예: 관계 / 일 / 사회 / 기술 / 기억

#### 주요 UX 포인트

- Threads처럼 태그 클릭으로 비슷한 글감 피드 이동
- 저장, 숨기기, 비슷한 것 더 보기
- “이 글감으로 바로 쓰기” CTA
- “학습 활동 먼저 하고 쓰기” 보조 CTA

#### 필터 구조

필터는 상단에 모두 펼치지 말고 **바텀시트**로 여는 것이 좋습니다. Material 3도 작은 화면에서 긴 옵션은 bottom sheet가 적합하다고 봅니다. ([Material Design][5])

필터 항목:

- 레벨 1/2/3
- 글 종류
- 길이
- 난이도
- 사고 유형
  설명 / 설득 / 비교 / 회고 / 비판 / 상상
- 주제군
  일상 / 사회 / 관계 / 기술 / 자기이해 / 진로 등

#### 추천 기능

- “최근 내가 잘 쓴 톤과 비슷한 글감”
- “최근 피한 주제와 다른 글감”
- “지금 5분 안에 쓸 수 있는 글감”
- “장문 에세이로 확장 가능한 글감”

이 탭은 결국 **글감 라이브러리**가 아니라 **글쓰기 진입 엔진**이어야 합니다.

### C. 글쓰기

이 탭은 이 제품의 심장입니다.
중요한 점은 “자유로운 에디터”라고 해도 실제 UX는 완전한 빈 화면보다 **가벼운 보조 구조를 가진 에디터**여야 한다는 점입니다.

#### 글쓰기 탭의 핵심 목적

- 자유 초안 작성
- 저장/재개
- AI 피드백과 수정
- 장문 글의 완성

#### 글 목록 페이지

1. 최상단 제목: "내 글"
2. CTA 버튼 카드:
   1. 왼쪽 컨텐츠 영역
      1. 제목: 글쓰기
      2. 설명: "새로운 이야기를 시작해보세요"
   2. 오른쪽 버튼 영역
      1. 글쓰기 아이콘
3. 글 목록
   1. 글 제목, 글 내용, 마지막 수정일, 글자 수
   2. 무한 스크롤 기반

#### 글쓰기 페이지

- 최상단 고정 헤더
  - 배경없이 플로팅 스타일
  - 왼쪽에 뒤로가기 아이콘 버튼
  - 오른쪽에 설정 톱니바퀴 아이콘 버튼
- 컨텐츠 영역
  - 제목: h1 요소와 contenteditable 속성 사용
    - placeholder: "어떤 이야기를 남길까요?"
  - 본문: tiptap 에디터 사용
    - placeholder: "생각이 흐르는 대로 자유롭게 적어보세요..."
- 하단 고정 플로팅 툴바
  - undo,redo 아이콘 버튼
  - 글자 수 및 단어 수 표시
  - AI 버튼

#### 글쓰기 페이지의 AI 기능

1. 문장 교정: 사용자가 특정 문장을 선택(드래그)한 후 AI 버튼 클릭 -> 어휘 & 표현 개선 / 문장 명료화 / 문장 호흡 조정 기능을 툴바가 확장되는 형태로 표시 -> 기능 선택 시 에디터 페이지가 상단(에디터)과 하단(AI 제안 결과)이 1:1 비율로 스플릿 뷰로 나뉨.(에디터 화면 위에 AI 제안 결과가 오버레이로 뜨는 것이 아님)
   - AI 제안 결과는 최대 3개의 제안이 표시됨. 적용 버튼을 누르면 원문이 제안으로 교체됨.
   - 스플릿 뷰로 전환될 때 및 다시 에디터 뷰로 돌아올 때 에디터의 스크롤이 유지되어야 함(사용자 경험을 위해 매우 중요)
2. 문서 피드백: 사용자가 문장 선택 없이 AI 버튼 클릭 -> 전체 맞춤법 & 중복 검토 / 문단 연결 흐름 피드백 기능을 툴바가 확장되는 형태로 표시 -> 기능 선택 시 AI 분석 결과가 에디터에 인라인 하이트라이트되고. 하단 툴바가 "검토 중", "종료" 버튼으로 상태 변경. 에디터에서 해당 하이라이트 클릭 시 피드백, 교체 제안이 표시됨.
