## 목표

사용자가 글 목록 또는 글쓰기 진입점에서 새 글을 만든 뒤, 막막함보다 시작 가능성을 더 크게 느끼도록 만든다.

## 범위

- 포함: 새 글 만들기, 제목 입력, 본문 첫 문장 작성, 임시저장 상태 확인
- 제외: 글감 기반 자동 채우기, 공개 전 검토, AI 피드백

## 시작과 종료

- 시작 조건: 사용자가 글쓰기 탭 또는 홈의 작성 CTA를 통해 새 글 생성 의도를 가진다.
- 종료 조건: 제목과 본문 초고가 입력되고 저장 상태가 인지된 상태가 된다.

## 주요 감정과 장벽

- 완전히 빈 편집기는 첫 문장을 쓰기 어렵게 만든다.
- 저장 여부가 보이지 않으면 길게 쓰기 전에 불안이 생긴다.
- 도구가 많아 보이면 작성보다 조작에 시선이 쏠린다.

## 핵심 의사결정 지점

- 새 글을 즉시 시작할지
- 제목부터 쓸지 본문부터 쓸지
- 초고를 짧게 저장해두고 나중에 이어 쓸지

## 흐름 요약

1. 사용자는 글 목록의 새 글 쓰기 CTA를 선택한다.
2. 에디터에 진입해 제목 또는 본문 중 편한 곳부터 입력한다.
3. 기본 툴 노출은 최소화된 상태에서 첫 문단에 집중한다.
4. 저장 상태가 자연스럽게 표시되어 초고를 남겨둘 수 있다는 확신을 얻는다.
