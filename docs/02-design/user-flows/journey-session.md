## 목적

여정 세션 진행은 글필의 핵심 학습 루프다. 구조화된 커리큘럼(여정 → 세션 → 스텝)을 통해 단계별 글쓰기 역량 성장을 경험한다.

## 시작 조건

- 사용자가 나의 여정 또는 홈에서 여정을 선택한 상태
- 진행 중인 세션이 있거나 여정을 처음 시작하는 상태

## 종료 조건

- 세션의 마지막 스텝을 완료하고 여정 상세 화면으로 복귀한 상태

## 핵심 사용자 목표

"오늘 10~20분 안에 하나의 학습 단위를 완료하고 싶다."

---

## 주요 흐름 (Happy Path)

```
홈 / 나의 여정 > 여정 카드 탭
  ↓
여정 상세 화면
  ├── 여정 설명 확인
  ├── 전체 진행률 카드 확인
  └── "지금 시작하기 →" 버튼 탭 (IN_PROGRESS 세션)
  ↓
세션 진행 플로우 (전체 화면)
  ├── 상단 프로그레스 바 (스텝 1/N)
  │
  ├── [LEARN] 개념 설명 읽기 → "다음"
  │
  ├── [GUIDED_QUESTION] 질문 답변 입력 → "답변 제출"
  │
  ├── [WRITE] 에세이 작성 → "제출"
  │     ↓
  │   AI 분석 중... ("글을 분석하고 있습니다")
  │     ↓
  ├── [FEEDBACK] AI 피드백 확인
  │     ├── 💚 강점 확인
  │     ├── 💡 개선점 확인
  │     ├── ❓ 사고 촉발 질문 확인
  │     └── "수정하기" 탭 (권장)
  │           ↓
  └── [REVISE] 피드백 참조 수정 → "재제출"
        ↓
      개선 피드백 ("문단 구조가 명확해졌습니다")
        ↓
세션 완료 애니메이션 "세션 완료!"
  └── 다음 세션 잠금 해제 안내
  ↓
여정 상세 화면 (진행률 업데이트)
```

---

## 분기 흐름

### AI 피드백에서 "다음" 선택 (수정 없이 진행)

```
[FEEDBACK] > "다음" 탭
  ↓
다음 스텝 또는 세션 완료
```

### 세션 중 앱 종료 후 재진입

```
앱 재실행 → 여정 상세 화면
  ↓
마지막 완료된 스텝부터 자동 재개
  └── 이전에 입력한 내용 보존
```

### AI 분석 실패

```
[WRITE] 제출 → AI 타임아웃 (10초 초과)
  ↓
"분석에 시간이 조금 더 필요합니다. 다시 시도해볼까요?"
  └── 재시도 버튼
```

### 마지막 세션 완료

```
세션 완료 애니메이션
  ↓
여정 완료 축하 화면 (전체 화면)
  ├── 여정 완료 배지 (P2)
  └── "다른 여정 찾기" CTA
```

## 핵심 장벽

| 장벽                     | 설계 해결책                                         |
| ------------------------ | --------------------------------------------------- |
| 세션이 너무 길게 느껴짐  | 상단 프로그레스 바로 진행 위치 항상 표시            |
| 피드백이 부담스러움      | 입문자는 강점 비율 2:1로 격려 중심                  |
| 수정하기 귀찮음          | "수정하기"를 primary CTA로 배치, "다음"은 secondary |
| 앱 종료 후 처음부터 다시 | 스텝 단위 자동 저장 + 재진입 시 이어서              |

## 결정 지점

- **FEEDBACK에서 수정 vs 스킵:** "수정하기"를 primary, "다음"을 secondary로 배치
- **REVISE에서 재제출 vs 그냥 넘어가기:** 재제출을 강하게 유도 (버튼 레이블, 위치)
