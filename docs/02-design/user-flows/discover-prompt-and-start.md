## 목표

주제가 막막한 사용자가 탐색 과정을 통해 자신에게 맞는 글감을 발견하고, 망설임 없이 작성 화면으로 이동하게 만든다.

## 범위

- 포함: 글감 목록 진입, 검색, 필터, 저장, 상세 확인, 바로 쓰기
- 제외: 홈 추천 로직, 학습 활동 연결, 공개 설정

## 시작과 종료

- 시작 조건: 사용자가 글감 찾기 화면 또는 홈의 글감 관련 CTA를 통해 탐색에 진입한다.
- 종료 조건: 사용자가 특정 글감을 선택하고 에디터에서 초고 작성을 시작한다.

## 주요 감정과 장벽

- 선택지가 너무 많으면 오히려 쓰기 시작이 늦어진다.
- 검색 결과가 차갑고 건조하게 보이면 감정적 연결이 약해진다.
- 저장과 바로 쓰기의 의미 차이가 명확하지 않으면 행동이 분산된다.

## 핵심 의사결정 지점

- 검색과 필터를 사용해 범위를 좁힐지
- 마음에 드는 글감을 저장만 할지
- 상세를 읽고 바로 쓰기로 넘어갈지

## 흐름 요약

1. 사용자는 글감 찾기에서 검색, 필터, 주제 칩으로 후보를 좁힌다.
2. 카드에서 분위기와 난이도를 보고 몇 개의 글감을 비교한다.
3. 필요하면 저장하고, 더 확신이 필요하면 상세를 연다.
4. 글감 상세에서 바로 쓰기를 누르면 글감 id를 포함하여 글을 서버에 생성(POST /writings)하고, 응답으로 받은 id의 `/writing/{id}` 에디터 페이지로 이동한다.
