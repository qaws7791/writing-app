## 목적

온보딩은 사용자의 관심사와 글쓰기 수준을 파악하여 글감, 여정, 피드백을 개인화하는 기반을 만든다. 진입 장벽을 최소화하고 최대한 빨리 첫 번째 글쓰기를 시작하도록 유도한다.

## 시작 조건

- 앱을 최초 설치하고 실행한 사용자

## 종료 조건

- 홈 화면에 진입 (관심사 기반 글감과 여정이 표시된 상태)

## 핵심 사용자 목표

"부담 없이 시작하고, 나에게 맞는 글쓰기를 빠르게 찾고 싶다."

---

## 흐름

```
앱 설치 후 최초 실행
  ↓
온보딩 화면
  ├── 관심 분야 선택 (마음챙김 / 글쓰기 기술 / 기술 등 다중 선택)
  ├── 글쓰기 경험 선택 (입문 / 중급 / 상급)
  └── "시작하기" 버튼
  ↓
회원가입 화면 (선택적 — 체험 먼저 가능)
  ├── Google / Apple 소셜 로그인
  └── "나중에 가입하기" (비회원 체험)
  ↓
홈 화면 (개인화된 글감 + 여정 추천)
```

---

## 비회원 체험 흐름

```
온보딩 완료 → "나중에 가입하기" 선택
  ↓
글감 탐색 → 글 에디터에서 글 작성
  ↓
"저장" 버튼 탭
  ↓
회원가입 유도 화면
  └── "가입하면 글이 저장됩니다."
  ↓
회원가입 완료 → 글 저장 → 서재에 추가
```

---

## 핵심 장벽

| 장벽               | 설계 해결책                                                     |
| ------------------ | --------------------------------------------------------------- |
| 회원가입 귀찮음    | 온보딩 → 체험 글쓰기 → 저장 시 회원가입 유도 (후순위)           |
| 선택지가 너무 많음 | 관심 분야는 3~5개 카테고리로 단순화                             |
| 수준 선택이 어색함 | "글쓰기 경험 거의 없음 / 가끔 씀 / 자주 씀" 등 행동 기반 레이블 |

## 결정 지점

- **회원가입 vs 체험 먼저:** 체험을 먼저 허용하고 저장 시 회원가입 유도
- **관심사 미선택:** 기본값(마음챙김)으로 진행
