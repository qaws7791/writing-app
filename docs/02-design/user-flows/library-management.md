## 목적

서재는 글쓰기 탭 내에서 사용자의 모든 글을 아카이빙하는 공간이다. 과거 글을 다시 읽고, 검색하고, 편집하며 성장의 궤적을 확인하는 흐름을 설계한다.

## 시작 조건

- 사용자가 하단 탭의 "글쓰기" 탭을 탭한 상태
- 글이 1개 이상 저장된 상태

## 종료 조건

- 특정 목표 행동 완료 (글 읽기 / 편집 시작)

## 핵심 사용자 목표

"내가 쓴 글을 다시 보고 싶다 / 예전에 어떤 글감으로 썼는지 확인하고 싶다."

---

## 주요 흐름

### 글 목록 탐색

```
글쓰기 탭
  ↓
서재 (글 카드 목록, 최신순)
  └── 날짜, 제목, 본문 미리보기, 단어 수 확인
  ↓
글 카드 탭
  ↓
글 상세 화면 (읽기 뷰)
  ├── 제목, 저자명, 날짜 확인
  ├── Writing Prompt 접힘 카드 탭 (글감 연결 시)
  │     └── 글감 제목, 본문 확인
  └── 본문 읽기
```

### 글 검색

```
글쓰기 탭 > 검색 바 탭
  ↓
키워드 입력
  ↓
글 제목 + 본문에서 실시간 검색
  └── 결과 목록 표시
  ↓
글 카드 탭 → 글 상세 화면
```

### 글 편집

```
글쓰기 탭 > 글 카드 > 더보기(⋮) 탭
  └── "편집하기" 선택
  ↓
글 에디터 (기존 내용 채워진 상태)
  └── 수정 후 완료(✓) 탭 → 글 상세 화면 (업데이트)
```

### 글 삭제

```
글쓰기 탭 > 글 카드 > 더보기(⋮) 탭
  └── "삭제" 선택
  ↓
삭제 확인 다이얼로그
  ├── "삭제" → 글 삭제 + 목록 복귀
  └── "취소" → 닫기
```

## 핵심 장벽

| 장벽                            | 설계 해결책                               |
| ------------------------------- | ----------------------------------------- |
| 저장한 글이 많아 못 찾음        | 검색 기능 (제목+본문)                     |
| 예전 글감이 뭐였는지 기억 안 남 | Writing Prompt 접힘 카드로 글감 맥락 복원 |
