## 목표

사용자가 중단된 초안을 다시 열었을 때 현재 위치와 다음 행동을 빠르게 파악하고 곧바로 작성 흐름에 복귀하게 만든다.

## 범위

- 포함: 홈의 이어서 쓰기 진입, 글 목록에서 초안 선택, 초안 재진입, 저장 상태 인지
- 제외: 버전 비교, 공개 범위 조정, 퇴고 모드

## 시작과 종료

- 시작 조건: 작성 중 초안이 존재하는 사용자가 홈 또는 글 목록에 진입한다.
- 종료 조건: 사용자가 기존 초안을 열고 본문 입력을 다시 시작한다.

## 주요 감정과 장벽

- 마지막으로 어디까지 썼는지 기억이 흐릿할 수 있다.
- 저장 상태가 불안하면 다시 쓰기 전에 망설이게 된다.
- 초안 목록이 복잡하면 지금 이어야 할 글을 고르기 어렵다.

## 핵심 의사결정 지점

- 홈의 이어서 쓰기 카드로 바로 진입할지
- 글 목록에서 다른 초안을 선택할지
- 초안을 열기 전에 저장 상태와 마지막 수정 시점을 확인할지

## 흐름 요약

1. 사용자는 홈의 이어서 쓰기 카드 또는 글 목록에서 초안을 확인한다.
2. 제목, 마지막 수정 시점, 짧은 미리보기로 현재 작업 대상을 판단한다.
3. 초안을 열고 제목과 최근 문단을 확인하며 맥락을 회복한다.
4. 저장 상태가 안정적으로 보이는 것을 확인한 뒤 집필을 이어간다.
