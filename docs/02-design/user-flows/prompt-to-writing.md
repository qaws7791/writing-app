## 목적

글쓰기 시작 흐름은 글필의 Secondary 기능인 자유 글쓰기의 핵심 루프다. 글쓰기 진입 화면에서 글감 발견 기회를 제공하면서도 "직접 쓸게요"로 즉시 에디터에 진입할 수 있게 설계한다.

## 시작 조건

- 사용자가 글쓰기 탭에 있는 상태

## 종료 조건

- 글이 서재에 저장되고 글 상세 화면에 도달한 상태

## 핵심 사용자 목표

"오늘 뭔가를 쓰고 싶은데, 주제를 찾고 바로 시작하고 싶다."

---

## 주요 흐름 A — 글감 선택 후 글쓰기 (Happy Path)

```
글쓰기 탭
  ↓
"새 글 쓰기" 버튼 탭
  ↓
글쓰기 진입 화면 ("오늘 어떤 글을 써볼까요?")
  ├── 필터 칩으로 유형 탐색 (선택적)
  └── 글감 카드 목록 확인
  ↓
글감 카드 탭
  ↓
글 에디터 (글감 참조 카드 고정)
  ├── 제목 입력
  ├── 본문 작성 (5초 간격 자동 저장)
  └── 완료(✓) 버튼 탭
  ↓
글 상세 화면 (읽기 뷰)
  └── 서재에 저장 완료
```

## 주요 흐름 B — 직접 쓰기

```
글쓰기 탭
  ↓
"새 글 쓰기" 버튼 탭
  ↓
글쓰기 진입 화면
  ↓
"직접 쓸게요" 버튼 탭
  ↓
빈 글 에디터
  ├── "아이디어가 필요하신가요?" 버튼 표시 (본문 비어 있는 동안)
  ├── 제목 입력
  ├── 본문 작성 (5초 간격 자동 저장)
  └── 완료(✓) 버튼 탭
  ↓
글 상세 화면 (읽기 뷰)
  └── 서재에 저장 완료
```

---

## 분기 흐름

### 에디터 내 글감 추가 탐색

```
빈 에디터 > "아이디어가 필요하신가요?" 탭
  ↓
글감 선택 Bottom Sheet
  ├── 필터 칩으로 유형 탐색
  └── 글감 카드 리스트
  ↓
글감 카드 탭
  ↓
시트 닫힘 + 에디터 상단에 글감 참조 카드 고정
  ↓
글 작성 계속
```

### 비회원 사용자의 저장 시도

```
에디터 > 완료(✓) 탭
  ↓
회원가입 유도 화면
  └── "가입하면 글이 저장됩니다."
  ↓
회원가입 완료 → 글 저장 → 서재에 추가
```

### 오프라인 상태

```
에디터 > 작성 중
  ↓
로컬 자동 저장 유지
  ↓
온라인 복귀 → 자동 동기화
```

## 핵심 장벽

| 장벽                     | 설계 해결책                                                     |
| ------------------------ | --------------------------------------------------------------- |
| 빈 화면 앞의 막막함      | 글쓰기 진입 화면에서 글감 탐색 기회 제공                        |
| 글감 탐색이 강제됨       | "직접 쓸게요" 버튼으로 진입 화면 스킵 가능                      |
| 에디터 진입 후 소재 부족 | "아이디어가 필요하신가요?" → Bottom Sheet로 추가 탐색 경로 제공 |
| 초고 완벽함 강박         | 자동 저장, "제출은 완성이 아닌 시작" 프레이밍                   |
