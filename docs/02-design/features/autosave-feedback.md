## 목적

저장 자체보다 저장이 되고 있다는 확신을 제공해 긴장 없이 초고를 이어 쓰게 만든다.

## 사용자 문제

- 저장 여부가 불명확하면 긴 글을 쓰기 전에 불안이 생긴다.
- 오류 문구가 과하면 작성 흐름이 끊기고 사용자 탓처럼 느껴질 수 있다.

## 범위

- 포함: 저장 중, 저장 완료, 동기화 지연, 저장 실패의 상태 인지
- 제외: 버전 히스토리, 수동 저장 정책, 충돌 해결 UI

## 진입 조건

- 사용자가 에디터에서 입력을 시작했다.

## 핵심 상호작용

- 저장 피드백은 시선 분산이 적은 위치에 짧게 보여야 한다.
- 완료 피드백은 안도감을 주되 성과 메시지처럼 과장하지 않는다.
- 지연이나 실패 시에도 작성 가능 상태를 우선 보장해야 한다.

## 규칙과 제약

- 저장 상태 문구는 차분하고 비난이 없어야 한다.
- 실패 상황에서도 이미 입력한 내용이 남아 있다는 정보를 우선 전달한다.
- 동일한 상태 변화는 일관된 위치와 문체로 보여야 한다.

## 예외 상태

- 저장 중이 길어짐: 진행 중임을 알리되 경고처럼 보이지 않게 한다.
- 저장 실패: 재시도 가능성과 임시 보존 여부를 함께 안내한다.
- 오프라인: 연결이 복구되면 다시 동기화된다는 기대를 분명히 전달한다.
