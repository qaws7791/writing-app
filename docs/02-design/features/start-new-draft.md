## 목적

새 글 시작을 복잡한 설정 절차가 아니라 즉시 행동 가능한 진입점으로 만든다.

## 사용자 문제

- 새 글 생성 전에 선택지가 많으면 시작이 늦어진다.
- 빈 화면에 들어간 뒤 무엇부터 써야 할지 막막할 수 있다.

## 범위

- 포함: 새 글 쓰기 CTA, 새 초안 생성, 에디터 진입
- 제외: 템플릿 선택기, 공개 범위 설정, 카테고리 분류

## 진입 조건

- 사용자가 글 목록 또는 홈에서 새로운 작성 의도를 가진다.

## 핵심 상호작용

- 새 글 시작 CTA를 누르면 즉시 빈 초안을 서버에 생성(POST)하고, 응답으로 받은 id로 `/write/{id}` 에디터 페이지로 이동한다.
- 별도의 `/write/new` 경로는 존재하지 않는다. Notion·Google Docs처럼 문서 생성이 먼저 일어나는 방식이다.
- 진입 즉시 편집 가능한 상태가 되어야 한다.
- 제목과 본문 중 어느 쪽부터 시작해도 흐름을 막지 않아야 한다.

## 규칙과 제약

- 새 글 시작 전 별도 설정 단계는 두지 않는다.
- CTA 클릭 시 POST 요청으로 초안 생성 → 응답 id로 에디터 리다이렉트하는 단일 흐름을 따른다.
- 생성 직후의 화면은 작성 중심이며 탐색 요소를 최소화한다.
- 초안이 생성되었다는 사실은 저장 경험과 자연스럽게 연결되어야 한다.

## 예외 상태

- 생성 실패: CTA 클릭 후 API 오류 시 사용자에게 재시도 안내를 제공한다.
- 네트워크 지연: 생성 요청 중에는 로딩 상태를 표시한다.
