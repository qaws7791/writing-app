## 목적

탐색 중의 흥미를 잃지 않고 보존하면서도, 저장이 실제 글쓰기 행동과 분리되지 않도록 균형을 맞춘다.

## 사용자 문제

- 지금 쓰기 어렵더라도 나중을 위해 남겨두고 싶은 글감이 있다.
- 저장만 반복하고 쓰지 않는 패턴이 생기면 제품 가치가 약해진다.

## 범위

- 포함: 카드/상세에서 저장, 저장 상태 확인, 저장 목록 진입 연결
- 제외: 저장 글감 컬렉션 관리, 태그 편집, 추천 보관함 전략

## 진입 조건

- 사용자가 홈 추천 또는 글감 탐색 화면에서 글감을 보고 있다.

## 핵심 상호작용

- 저장 액션은 가볍고 즉시 이해되어야 한다.
- 저장 후에는 완료 피드백이 짧고 차분하게 보여야 한다.
- 저장이 쓰기를 대체하는 행동처럼 과도하게 강조되지 않아야 한다.

## 규칙과 제약

- 저장 CTA는 보조 액션으로 유지한다.
- 저장 여부는 카드와 상세에서 일관된 표기로 보인다.
- 저장 결과는 프로필 또는 홈의 저장한 글감 영역과 연결될 수 있어야 한다.

## 예외 상태

- 저장 실패: 사용자 탓처럼 보이지 않는 문구로 재시도를 안내한다.
- 중복 저장: 중복 행동을 오류로 다루지 않고 이미 저장됨 상태를 유지한다.
