## 목적

세션은 여정 내 하나의 학습 노드로, 10~20분 완결형으로 설계된다. 세션은 여러 스텝으로 구성되며, 각 스텝은 다른 유형의 학습 활동을 제공한다.

## 스텝 유형 정의

| 유형                  | 설명                | 주요 액션                   |
| --------------------- | ------------------- | --------------------------- |
| **LEARN**             | 개념 학습           | 읽기 → "다음"               |
| **READ**              | 예시 읽기           | 읽기 → "다음"               |
| **GUIDED_QUESTION**   | 가이드 질문 답변    | 텍스트 입력 → "답변 제출"   |
| **FILL_IN_THE_BLANK** | 빈칸 채우기         | 선택지 탭 → "완성 확인"     |
| **WRITE**             | 에세이 글쓰기       | 텍스트 입력 → "제출"        |
| **FEEDBACK**          | AI 피드백 확인      | 읽기 → "수정하기" or "다음" |
| **REVISE**            | 피드백 참조 후 수정 | 수정 → "재제출"             |

---

## 스텝별 상세 규칙

### LEARN (개념 학습)

**목적:** 이번 세션에서 다루는 핵심 개념을 설명
**UI:**

- 제목 + 설명 텍스트 + (선택적) 예시 카드
- "다음" 버튼 하단 고정

**규칙:**

- 내용을 읽지 않아도 즉시 "다음" 탭 가능 (강제 읽기 없음)
- REQ-SES-002

---

### READ (예시 읽기)

**목적:** 모범 에세이 또는 문단을 통해 학습
**UI:**

- 텍스트 블록 + 하이라이트된 분석 포인트
- "다음" 버튼 하단 고정

**규칙:**

- LEARN과 동일, 강제 읽기 없음

---

### GUIDED_QUESTION (가이드 질문)

**목적:** 메타인지를 촉발하는 열린 질문에 답하며 사고 구조화
**UI:**

- 질문 텍스트
- 텍스트 입력 필드 (여러 줄)
- "답변 제출" 버튼 (키보드 위 고정)

**규칙:**

- 최소 입력 제한 없음 (짧은 답변도 허용)
- 답변은 저장되어 이후 글쓰기 참조 가능
- REQ-SES-003

---

### FILL_IN_THE_BLANK (빈칸 채우기)

**목적:** 단어 선택을 통해 어휘 감각과 표현력 훈련
**UI:**

- 지시문: "빈칸에 들어갈 적절한 단어를 선택하세요"
- 문장 (빈칸 포함): "그는 [____] 눈빛으로 나를 바라보았다."
- 선택지 목록 (1~4개)
- "완성 확인" 버튼 하단 고정

**규칙:**

- 선택지 탭 시 즉시 선택 상태 강조
- "완성 확인" 탭 시 정답 여부 표시 후 다음 스텝

---

### WRITE (글쓰기)

**목적:** 세션 학습 내용을 바탕으로 실제 에세이 작성
**UI:**

- Writing Prompt 참조 카드 (접힘 상태)
- 제목 + 본문 에디터
- 단어 수 카운터 (실시간)
- "제출" 버튼 (키보드 위 고정)

**규칙:**

- 5초 간격 자동 저장 (REQ-SES-008)
- 최소 단어 수 요구 없음 (v1)
- "제출" 탭 시 AI 분석 시작
- 제출 후 최대 10초 이내 FEEDBACK 스텝으로 전환
- AI 분석 타임아웃/실패 시: 재시도 버튼 (REQ-AI-006)
- REQ-SES-004

---

### FEEDBACK (AI 피드백)

**목적:** AI 소크라테스식 코칭으로 글의 강점과 개선 방향 제시
**UI:**

- 제출된 글 (읽기 전용, 접힘 가능)
- AI 피드백 카드:
  - 💚 강점 (1~2개)
  - 💡 개선점 (1~2개)
  - ❓ 사고 촉발 질문 (1개)
- "수정하기" 버튼 (primary)
- "다음" 버튼 (secondary)

**규칙:**

- "수정하기" → REVISE 스텝 진입
- "다음" → 수정 없이 다음 스텝 또는 세션 완료
- "수정하기"를 더 눈에 띄게 배치 (디자인 원칙 3)
- REQ-SES-005

---

### REVISE (수정)

**목적:** 피드백을 참조하며 초안을 개선하는 퇴고 경험
**UI:**

- 피드백 카드 참조 (접힘 상태 또는 사이드 패널)
- 수정 가능한 에디터 (초안 텍스트 미리 채워진 상태)
- "재제출" 버튼 (키보드 위 고정)

**규칙:**

- 재제출 시 AI가 초안 대비 수정본 비교 분석
- "문단 구조가 명확해졌습니다" 등 구체적 개선 피드백 표시
- 개선된 부분을 명시적으로 언급 (REQ-AI-005)
- 재분석 후 다음 스텝 또는 세션 완료로 진행
- REQ-SES-006

---

## 상태 보존 규칙

- 세션 진행 중 앱 종료 시 마지막 완료 스텝까지 자동 저장 (REQ-SES-007)
- 재진입 시 해당 스텝부터 재개
- WRITE 스텝에서 입력한 내용은 별도 보존
