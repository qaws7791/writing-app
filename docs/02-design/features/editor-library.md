## 목적

에디터는 사용자가 에세이를 작성하는 핵심 공간이다. 서재는 작성한 모든 글의 아카이브다. 두 기능은 "글 작성 → 저장 → 아카이빙 → 재방문"의 루프를 구성한다.

## 에디터

### 핵심 기능

**제목 + 본문 입력**

- 큰 폰트 제목 필드 + 클린 본문 영역
- v1 서식: 볼드, 이탤릭 (최소)

**단어 수 카운터**

- 실시간 업데이트
- 표시 위치: 하단 또는 키보드 위
- REQ-EDT-001

**자동 저장**

- 5초 간격으로 백그라운드 저장
- 별도 UI 없음 (조용한 저장)
- 오프라인 시 로컬 저장, 온라인 복귀 시 자동 동기화
- REQ-SES-008, REQ-NFR-002

**완료(✓) 버튼**

- 탭 시 글을 서재에 저장하고 글 상세 화면으로 이동
- REQ-EDT-002

**Writing Prompt 참조 카드**

- 글감 연결 시 에디터 상단에 접힘 상태로 표시
- 탭 시 펼쳐져 글감 제목과 본문 확인 가능

### 키보드 대응

- 완료(✓) / 제출 버튼: `visualViewport API`로 키보드 위 고정
- Writing Prompt 카드: 키보드 활성화 시 접힘 유지
- CSS 높이: `height: 100dvh`

### 오프라인 상태

- 로컬에 임시 저장
- 조용한 "오프라인" 배너 표시
- 온라인 복귀 시 자동 동기화

## 서재

### 핵심 기능

**글 목록**

- 최신순 정렬 (기본)
- 각 카드: 날짜, 제목, 본문 미리보기(2줄), 단어 수, 더보기(⋮) 메뉴
- REQ-EDT-004

**검색**

- 검색 바에 키워드 입력 시 글 제목+본문에서 검색
- 결과 없음 시: "'{검색어}'에 대한 결과가 없어요."
- REQ-EDT-003

**더보기 메뉴 (⋮)**

- 편집하기 → 에디터로 이동
- 삭제 → 삭제 확인 다이얼로그

### Writing Prompt 연결

- 글 상세에서 Writing Prompt 접힘 카드 제공
- 펼치면 연결된 글감의 제목, 본문, "글감 상세 보기 →" 링크 표시
- REQ-EDT-005
