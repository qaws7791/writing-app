## 목적

작성자가 작성 이력을 신뢰할 수 있도록, 자동·수동 저장된 버전을 열람하고 원하는 시점으로 되돌릴 수 있게 한다.

## 사용자 문제

- 이전에 쓴 내용이 어떤 상태였는지 확인할 방법이 없으면 불안하다.
- 실수로 글을 크게 변경했을 때 되돌릴 수단이 필요하다.

## 범위

- 포함: 버전 목록 조회, 특정 버전 미리보기, 버전 복원, 수동 버전 저장
- 제외: 버전 간 diff 비교, 버전 삭제, 버전 이름 편집

## 진입 조건

- 에디터 화면에서 설정 메뉴 → "버전 기록" 클릭

## 핵심 상호작용

### 버전 목록 조회

- 모달이 열리면 서버에서 버전 목록을 불러온다.
- 각 항목에 버전 시각, 라벨(자동 저장 / 수동 저장 / 복원됨)을 표시한다.
- 최신 버전이 기본 선택 상태이며, 현재 에디터 내용을 미리보기한다.
- 로딩 중에는 스켈레톤을, 오류 시 재시도 안내를 보여준다.

### 버전 미리보기

- 목록에서 항목을 선택하면 해당 버전의 전체 내용을 미리보기 영역에 표시한다.
- 미리보기는 읽기 전용 prose 스타일로 렌더링한다.

### 버전 복원

- 선택한 버전에 "이 버전으로 복원" 버튼을 표시한다.
- 현재 버전에는 복원 버튼을 표시하지 않는다.
- 클릭 시 확인 다이얼로그를 보여준다.
- 복원 완료 후 모달을 닫고 에디터를 업데이트한다.

### 수동 버전 저장

- 에디터 헤더 설정 메뉴에서 "현재 버전 저장" 항목을 선택한다.
- 현재 내용을 즉시 서버에 push하면서 스냅샷을 생성한다.
- 완료 시 토스트로 확인 피드백을 준다.

## 종료 조건

- 모달 닫기 또는 복원 완료 후 에디터로 복귀

## 예외 처리

| 상황                | 대응                                              |
| ------------------- | ------------------------------------------------- |
| 버전 목록 로딩 실패 | "버전 기록을 불러올 수 없습니다" + 재시도 버튼    |
| 버전 상세 로딩 실패 | "이 버전을 불러올 수 없습니다" 인라인 오류        |
| 복원 실패           | "복원에 실패했습니다. 다시 시도해 주세요." 토스트 |
| 버전이 없음         | "아직 저장된 버전이 없습니다" 빈 상태 표시        |
| 네트워크 오프라인   | "오프라인 상태에서는 버전 기록을 볼 수 없습니다"  |
