## 목적

주제가 막막한 상태를 탐색 가능한 상태로 바꾸고, 선택 피로를 낮춘다.

## 사용자 문제

- 후보가 너무 많으면 쓰기보다 탐색만 오래 하게 된다.
- 탐색 기준이 불분명하면 자신에게 맞는 글감을 찾기 어렵다.

## 범위

- 포함: 검색, 필터, 주제 칩, 목록 비교, 상세 진입
- 제외: 추천 피드 알고리즘, 학습 콘텐츠 탐색, 커뮤니티 인기 글감

## 진입 조건

- 사용자가 글감 찾기 화면에 진입한다.
- 검색 또는 필터를 사용하지 않아도 기본 목록이 존재한다.

## 핵심 상호작용

- 검색은 즉시 범위를 좁히는 용도로 동작해야 한다.
- 필터는 길이와 주제군 중심의 최소 집합으로 유지한다.
- 주제 칩은 빠른 탐색 보조 장치이며 필터보다 가볍게 인지되어야 한다.
- 카드 비교에서 정보 밀도가 과하지 않아야 한다.

## 규칙과 제약

- 탐색 화면은 읽기보다 선택을 돕는 구조여야 한다.
- 검색 결과가 없어도 막다른 화면처럼 보이지 않아야 한다.
- 저장과 상세/바로 쓰기 CTA가 경쟁하지 않도록 우선순위를 유지한다.

## 예외 상태

- 검색 결과 없음: 검색어 수정 제안과 추천 주제 칩을 함께 보여준다.
- 필터 결과 없음: 필터 해제 CTA를 분명히 제공한다.
- 네트워크 오류: 목록 대신 재시도와 홈/저장 글감 이동 대안을 제시한다.
