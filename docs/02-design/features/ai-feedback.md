## 목적

AI는 완성된 텍스트를 생성하지 않는다. 대신 **진단자, 질문자, 격려자, 안내자** 역할을 수행하여 사용자 스스로 사고하고 성장하도록 돕는다.

이 원칙은 [[02-design/design-principles]] "원칙 1 — 사고 우선"에서 직접 파생된다.

## AI의 역할 정의

| 역할       | 설명                          | 예시                                                                             |
| ---------- | ----------------------------- | -------------------------------------------------------------------------------- |
| **진단자** | 글의 구조적/논리적 특성 분석  | "이 글은 3개의 문단으로 구성되어 있고, 중심 논지가 두 번째 문단에서 전환됩니다." |
| **질문자** | 사고를 확장하는 질문 제시     | "이 주장에 동의하지 않는 사람은 어떤 반론을 펼칠까요?"                           |
| **격려자** | 개선된 부분을 구체적으로 칭찬 | "초안보다 근거가 구체적이 되었고, 특히 두 번째 예시가 설득력을 높였습니다."      |
| **안내자** | 다음 단계를 위한 힌트 제공    | "결론에서 서론의 질문을 다시 한번 떠올려 보는 건 어떨까요?"                      |

## 피드백 구조

```
AI 피드백 카드
├── 💚 강점 (1~2개)
│   └── 구체적으로 잘된 부분과 이유
├── 💡 개선점 (1~2개)
│   └── 문제 지적 + 방향 힌트 (답을 주지 않음)
└── ❓ 사고 촉발 질문 (1개)
    └── 글의 깊이를 확장하는 메타인지 질문
```

## 수준별 피드백 전략

| 수준 | 강점 : 개선점 비율 | 피드백 깊이      | 메타인지 질문 유형                                           |
| ---- | ------------------ | ---------------- | ------------------------------------------------------------ |
| 입문 | 2 : 1              | 문장/문단 단위   | "이 문장에서 가장 하고 싶은 말은?"                           |
| 중급 | 1 : 1              | 논리 구조 단위   | "이 근거가 주장을 충분히 뒷받침하나요?"                      |
| 상급 | 1 : 2              | 에세이 전체 단위 | "이 글의 전제에 동의하지 않는 독자를 어떻게 설득하시겠어요?" |

## 재제출 시 비교 피드백

- 초안 대비 수정본의 개선된 부분을 명시적으로 언급
- 예시: "문단 구조가 명확해졌습니다", "특히 두 번째 예시가 설득력을 높였습니다"
- REQ-AI-005

## AI가 하지 않는 것

- 완성된 문장이나 문단을 생성하지 않는다
- 대체 표현을 직접 제시하지 않는다
- 맞춤법/문법 교정을 자동으로 적용하지 않는다 (지적은 하되, 수정은 사용자 몫)
- 글의 주제나 방향을 결정하지 않는다

REQ-AI-003

## 성능 요구사항

- 제출 후 10초 이내 피드백 생성 완료 (REQ-AI-001)
- 타임아웃 또는 실패 시: "분석에 시간이 조금 더 필요합니다. 다시 시도해볼까요?" + 재시도 버튼 (REQ-AI-006)
