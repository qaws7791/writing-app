## 목적

에디터를 기능 전시 공간이 아니라 생각을 문장으로 옮기는 조용한 작업 공간으로 유지한다.

## 사용자 문제

- 편집 도구가 많아 보이면 집필 흐름이 끊긴다.
- 모바일에서는 긴 글 구조를 한 번에 파악하기 어렵다.

## 범위

- 포함: 제목 입력, 본문 입력, 최소 툴바, 뒤로 가기, 설정 진입, 선택되지 않은 기본 작성 상태
- 제외: AI 보조 상세, 퇴고 모드, 공개 설정, 협업 편집

## 진입 조건

- 새 글 시작 또는 기존 글 재진입으로 에디터를 연다.

## 핵심 상호작용

- 첫 입력 지점이 명확해야 한다.
- 기본 툴 노출은 초고 작성에 필요한 수준으로 제한한다.
- 작성 중 화면은 읽기와 쓰기 리듬을 해치지 않아야 한다.

## 규칙과 제약

- 상단과 하단의 보조 UI는 떠 있지만 주의를 빼앗지 않아야 한다.
- 제목과 본문은 시각적으로 명확히 구분되어야 한다.
- 작성 상태의 주 피드백은 저장 안정감과 입력 연속성에 집중한다.

## 예외 상태

- 빈 글: 부담을 낮추는 기본 힌트를 제공하되 과한 안내는 피한다.
- 긴 글: 현재 위치를 잃지 않도록 스크롤과 레이아웃 안정성을 유지한다.
- 오프라인 또는 저장 지연: 작성 가능 상태는 유지하면서 저장 상태를 알려준다.
