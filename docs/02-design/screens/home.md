## 목적

홈은 사용자가 앱을 열었을 때 즉시 글을 쓸 수 있도록 글쓰기 진입을 최우선으로 배치하고, 진행 중인 여정을 그 아래에서 확인할 수 있도록 설계된 대시보드다. 서브탭 없이 단일 구조로 인지 부담을 낮춘다.

> **설계 의도:** 홈에서 글감을 직접 노출하지 않는다. 글감 탐색은 글쓰기 탭 진입 시 중간 화면에서 이루어지며, 홈에서의 노출은 글감을 핵심 기능처럼 보이게 하는 잘못된 위계를 만든다.

## 진입 경로

- 앱 최초 진입
- 하단 탭 바 "홈" 탭 탭

## 화면 구성

**인사 영역**

- 시간대 기반 동적 인사: "좋은 아침입니다, {이름}님."
- 텍스트 크기가 크고, 화면 상단 좌측 배치

**[PRIMARY] 오늘의 글쓰기 제안**

- 인사 영역 바로 아래, 여정 목록보다 상단에 배치
- "오늘 자유롭게 글을 써보세요" + 새 글쓰기로 이동하는 버튼
- 글감 제목을 직접 노출하지 않고, 글쓰기 공간으로의 진입만을 유도
- `showWritingSuggestion` 조건이 true일 때만 표시

**[PRIMARY] 나의 여정 목록**

- 글쓰기 제안 아래에 배치
- 진행 중인 모든 여정 카드 리스트
- 각 카드에 썸네일, 여정명, 현재 세션명, 진행률 포함
- "여정 더보기 →" 링크 → 여정 탭으로 이동

## 상태별 UI

### 진행 중인 여정 없음

- "현재 진행 중인 여정 카드" 대신 "첫 여정 시작하기" CTA를 동일 위치에 표시
- 여정 탭으로 이동하는 버튼 포함

### 로딩 상태

- 카드 영역에 스켈레톤 표시. 인사 영역은 즉시 렌더링.

### 오프라인 상태

- 캐시된 여정 카드를 표시. 상단에 "오프라인 모드" 배너 표시.
