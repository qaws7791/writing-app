## 목적

세션 진행 플로우는 사용자가 여정의 한 세션을 처음부터 끝까지 완주하는 전체 화면 경험이다. 하단 탭 바를 숨기고 스텝별 집중 경험을 제공한다.

## 진입 경로

- 여정 상세 > "지금 시작하기 →" 탭
- 세션 중 앱 종료 후 재진입 시 마지막 완료 스텝부터 재개

## 공통 UI 요소

**상단 프로그레스 바**

- `position: sticky` 유지 (키보드 활성화 시에도)
- 현재 스텝 / 전체 스텝 수 시각화
- 뒤로 가기 (←) — 여정 상세로 복귀
- 더보기 메뉴 (⋮)

**탭 바**

- 세션 진행 중 완전히 숨김

## 스텝 유형별 UI

### 개념 학습 (Learn)

**구성:**

- 제목 (예: "논증의 기본 구조")
- 핵심 개념 설명 텍스트
- 예시 카드 (선택적)

**액션:**

- "다음" 버튼 (하단 고정) → 다음 스텝으로 전환

---

### 예시 읽기 (Read)

**구성:**

- 모범 에세이 또는 문단
- 분석 포인트 하이라이트

**액션:**

- "다음" 버튼 (하단 고정) → 다음 스텝으로 전환

---

### 가이드 질문 (Guided Question)

**구성:**

- 메타인지 촉발 질문 (예: "이 글을 통해 해결하고자 하는 핵심 문제는 무엇인가요?")
- 텍스트 입력 필드

**액션:**

- "답변 제출" 버튼 (키보드 위 고정) → 답변 저장 후 다음 스텝

---

### 빈칸 채우기 (Fill-in-the-blank)

**구성:**

- 지시문: "빈칸에 들어갈 적절한 단어를 선택하세요"
- 문장 (빈칸 포함): 예: "그는 [____] 눈빛으로 나를 바라보았다."
- 선택지 목록 (1, 2, 3, 4번)

**액션:**

- 선택지 탭 → 선택 상태 표시
- "완성 확인" 버튼 → 정답 확인 후 다음 스텝

---

### 글쓰기 (Write)

**구성:**

- 세션 글감/프롬프트 참조 카드 (접힘 상태)
- 풀스크린 에디터 (제목 + 본문)
- 단어 수 카운터

**키보드 대응:**

- "제출" 버튼: 키보드 위 고정 (`visualViewport API`)
- Writing Prompt 참조 카드: 접힘 유지

**액션:**

- "제출" 탭 → AI 분석 시작 → AI 피드백 스텝으로 전환

**제출 후 상태:**

- 로딩: "글을 분석하고 있습니다" 메시지 + 인디케이터
- 10초 이내 AI 피드백 스텝 전환
- 실패: "분석에 시간이 조금 더 필요합니다. 다시 시도해볼까요?" + 재시도 버튼

---

### AI 피드백 (Feedback)

**구성:**

- 제출된 글 (읽기 전용, 접힘 가능)
- AI 피드백 카드:
  - 💚 강점 (1~2개): 구체적으로 잘된 부분과 이유
  - 💡 개선점 (1~2개): 방향 힌트 (답 주지 않음)
  - ❓ 사고 촉발 질문 (1개): 깊이를 확장하는 메타인지 질문

**수준별 피드백 비율:**

- 입문: 강점 2 : 개선점 1
- 중급: 강점 1 : 개선점 1
- 상급: 강점 1 : 개선점 2

**액션:**

- "수정하기" 버튼 (primary) → 수정(Revise) 스텝
- "다음" 버튼 (secondary) → 수정 없이 다음 스텝 또는 세션 완료

---

### 수정 (Revise)

**구성:**

- 피드백 카드 (접힘/사이드 패널 — 참조 가능)
- 인라인 피드백이 표시된 에디터

**액션:**

- "재제출" 버튼 (키보드 위 고정) → AI 비교 분석
- 재제출 후: "문단 구조가 명확해졌습니다" 등 개선 피드백 표시
- 다음 스텝 또는 세션 완료로 진행

---

## 세션 완료

- 완료 축하 애니메이션 표시
- "세션 완료!" 메시지
- 다음 세션 잠금 해제 안내
- 여정 상세로 복귀 버튼

---

## 상태 보존

- 세션 진행 중 앱이 종료되면 마지막 완료 스텝까지 자동 저장
- 재진입 시 해당 스텝부터 재개
- 글쓰기 스텝에서 5초 간격으로 자동 저장
