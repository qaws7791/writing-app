## 목적

에디터는 산만함을 제거하고 사용자가 글쓰기 흐름에 집중할 수 있는 화면이다. 글감에서 시작하거나 새 글로 시작할 수 있다.

## 진입 경로

- 글감 상세 > "글쓰기 시작" 탭
- 세션 플로우 > 글쓰기(Write) 스텝

## 화면 구성

**헤더**

- 뒤로 가기 (←)
- 글 제목 (편집 가능, 중앙 또는 제목 필드와 연동)
- 더보기 메뉴 (⋮)
- 완료 버튼 (✓, 우측) — 탭 시 글을 저장하고 글 상세로 이동

**날짜 표시**

- 작성일 (예: "2024년 6월 12일 수요일")

**제목 필드**

- 큰 폰트 (H1 수준)
- 플레이스홀더: "제목"
- 편집 가능

**본문 영역**

- 클린 텍스트 입력 영역
- 플레이스홀더 없음 — 빈 여백이 초대
- 산세리프 폰트, 적절한 줄 간격 (1.7~1.8)
- v1 서식 기능: 볼드, 이탤릭 (최소)

**Writing Prompt 참조 카드** (글감 연결 시)

- 에디터 상단 또는 키보드 위에 접힘(collapsed) 상태로 표시
- 탭 시 펼쳐져 글감 제목과 본문 확인 가능
- 글쓰기 영역 확보를 위해 기본 접힘 상태 유지

**단어 수 카운터**

- "{N} 단어" — 실시간 업데이트
- 화면 하단 또는 키보드 위 고정

---

## 자동 저장

- 5초 간격으로 자동 저장
- 저장 중 별도 UI 없음 (조용한 저장)
- 저장 실패 시: "저장에 실패했어요. 잠시 후 다시 시도해주세요."

---

## 키보드 대응

- 키보드 활성화 시:
  - 헤더(프로그레스 바 포함) — `position: sticky` 유지
  - 완료(✓) 또는 제출 버튼 — 키보드 위 고정 (`visualViewport API` 활용)
  - Writing Prompt 참조 카드 — 접힘 상태 유지로 입력 영역 최대 확보
- CSS 높이 기준: `height: 100dvh`

---

## 상태별 UI

### 기본 상태 (자유 글쓰기)

- 제목 + 본문 영역만 표시. 탭 바는 숨김.
- 완료(✓) 버튼이 항상 헤더 우측에 표시.

### 글감 연결 상태

- Writing Prompt 카드가 접힘 상태로 상단에 표시.
- 본문 입력 영역이 메인 포커스.

### 세션 글쓰기 스텝

- 세션 플로우 내에서 에디터가 활성화됨.
- "완료(✓)" 대신 "제출" 버튼.
- 상단에 세션 프로그레스 바 유지.
- 참고: [[02-design/screens/session-flow]]

### 오프라인 상태

- 로컬에 임시 저장. 온라인 복귀 시 자동 동기화.
- 상단에 조용한 "오프라인" 표시.
