## 목적

에디터는 산만함을 줄이고 사용자가 초고 흐름을 잃지 않게 만드는 작업 화면이다.

## 진입 경로

- 홈의 추천 글감 또는 이어서 쓰기
- 글감 상세의 바로 쓰기
- 글 목록의 새 글 쓰기 또는 초안 선택

## 범위

- 포함: 제목 입력, 본문 입력, 플로팅 헤더, 최소 툴바, 저장 상태 피드백
- 제외: AI 기능 상세 플로우, 퇴고 모드, 공개 설정, 협업 기능

## 정보 위계

1. 제목
2. 본문 입력 영역
3. 저장 상태
4. 보조 툴바와 설정

## 주요 액션

- 제목 입력
- 본문 입력
- 뒤로 가기
- 설정 진입

## 상태별 UI

### 기본 상태

- 제목 입력과 본문 입력이 명확히 구분된다.
- 상단 헤더와 하단 툴바는 떠 있지만 시선을 빼앗지 않는다.
- 저장 상태는 화면의 조용한 위치에서 짧게 인지된다.

### 빈 초안 상태

- 제목 또는 첫 문장을 시작할 수 있는 가벼운 힌트를 제공한다.
- 힌트는 장황하지 않고 입력 전 불안을 낮추는 수준에 그친다.

### 글감 기반 진입 상태

- 선택한 글감의 흔적이 과하지 않게 보조 맥락으로 남아 있어야 한다.
- 사용자의 본문 입력이 항상 더 큰 시각적 우선순위를 가진다.

### 저장 중 상태

- 입력은 계속 가능해야 하며 저장 중 표시가 경고처럼 보이지 않아야 한다.

### 저장 완료 상태

- 짧고 안정적인 완료 피드백만 보여준다.

### 저장 지연 또는 실패 상태

- 작성 내용이 남아 있다는 사실을 먼저 전달한다.
- 연결 복구 또는 재시도 기대를 차분하게 안내한다.

## 문구 톤 가이드

- 저장 관련 문구는 사용자 탓으로 읽히지 않아야 한다.
- 예시: "작성 내용은 임시 저장되어 있습니다."

## 네비게이션 이동

- 뒤로 가기: [[02-design/screens/writing-list]] 또는 직전 진입 화면
- 설정: 에디터 내부 보조 패널 또는 별도 후속 화면

## 데스크톱 차이

- 본문 폭과 여백을 더 넓게 가져갈 수 있다.
- 보조 패널이나 메타 정보는 추가할 수 있지만 기본 작성 흐름 우선순위는 동일해야 한다.
